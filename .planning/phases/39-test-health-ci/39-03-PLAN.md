<objective>
Add 7-day throttled upstream version check to gsd-check-update.js so CI detects when upstream publishes a new version without checking every session.

Purpose: Currently the hook has a 4-hour TTL: if the cache is less than 4 hours old, it skips the background spawn entirely. This means on a busy day with many sessions, the npm registry or git upstream is checked multiple times. Per D-05, extend this with a 7-day "full check" throttle: the 4-hour TTL remains as the fast-path skip (avoids spawning a background process at all), but when the background process does run, it first checks if the cache's `checked` timestamp is less than 7 days old. If so, the background process reuses the existing cache data without making a network request. This is the decorator pattern: the 7-day throttle wraps the existing check logic without modifying it.

Per D-06: Client-side polling with time-based throttle. No GitHub Actions dependency.
Per D-07: If check fails (network error), silently skip and retry next session. Existing behavior already handles this.

Output: Updated hook with 7-day throttle, tests for throttle behavior
</objective>

    Layer 2 (NEW): 7-day throttle in the BACKGROUND PROCESS (inside the spawn string, lines 59-186).     - 0-4h: No subprocess spawned (layer 1)
