<objective>
Eliminate flaky subprocess timeout failures by creating a central timeout constant and applying it consistently across sync.test.cjs and hooks.test.js.

Purpose: Multiple subprocess-heavy tests in sync.test.cjs use the 5000ms bun default timeout, which is insufficient on Windows where git subprocess operations take longer. Similarly, hooks.test.js maintainer path tests use inline { timeout: 15000 } or the 5000ms default inconsistently. Per D-03, the project convention from Phase 37 is 15000ms for subprocess-heavy tests. Rather than applying Shotgun Surgery (adding { timeout: 15000 } to each test individually), create a single constant that all subprocess tests import. This follows OCP: new subprocess tests automatically get the right timeout by importing the constant, and changing the timeout value requires exactly one edit.

Per D-04: Also evaluate sync-preview timeout tests for Windows-specific issues and increase timeout as needed.

Output: Central timeout constants, updated test files, reliable test passes on Windows
</objective>

        * Central timeout constants for subprocess-heavy tests.

        * Windows subprocess operations (git init, git commit, node spawns) take
        * significantly longer than macOS/Linux. These constants prevent flaky
        * timeout failures without per-test hardcoding.

        * Usage: const { SUBPROCESS_TIMEOUT } = require('./helpers');

       module.exports = { SUBPROCESS_TIMEOUT, HEAVY_SUBPROCESS_TIMEOUT };
