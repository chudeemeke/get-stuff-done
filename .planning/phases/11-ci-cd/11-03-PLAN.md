  <name>Task 1: Tests for hook scripts</name>
  <files>tests/hooks.test.js</files>
  <action>
    Create tests/hooks.test.js covering all 3 hook scripts.

    **Testing strategy for hooks:** Hooks are CLI scripts (#!/usr/bin/env node) without module.exports. They read stdin, access filesystem, and write to stdout. Test them via two approaches:
    1. **CLI execution tests:** Run the hook as a child process with controlled env/args and verify stdout/stderr/exit code
    2. **Internal logic extraction tests:** Where hooks have extractable pure functions, require the file in a controlled environment with mocked globals

    For approach 1, use execSync with env overrides. For approach 2, consider that requiring a file with a top-level main() call will execute it -- so use env vars as gates or mock process.argv to control execution flow.

    **gsd-check-update.js tests:**
    - Reads existing cache file and outputs cached version info
    - Creates cache directory if it does not exist
    - Handles missing/corrupt cache file gracefully (no crash)
    - Handles network unavailability gracefully (mock execSync for npm view to throw)
    - Writes cache with timestamp for TTL checking

    **gsd-statusline.js tests:**
    - Produces output string containing expected segments (model, directory info)
    - Reads .planning/STATE.md for project state when available
    - Handles missing .planning/ directory gracefully
    - Handles missing config file gracefully (uses defaults)
    - Output contains ANSI codes when terminal supports color
    - Reads stdin input (Claude Code passes context via stdin)

    **pre-compact.js tests:**
    - Reads .planning/ directory for state preservation
    - Creates state snapshot before compaction
    - Handles missing .planning/ directory (no crash, clean exit)
    - Writes preservation data to expected location
    - Reads stdin input (Claude Code passes context via stdin)

    **Mocking approach:**
    - Set GSD_HOME and GSD_CONFIG_PATH env vars to temp directories
    - Create minimal .planning/ structures in temp dir using helpers from Plan 01
    - For stdin-reading hooks: Use execSync with input option to pipe test data
    - For filesystem hooks: Set cwd to temp dir with controlled structure
    - Mock npm registry calls by setting env vars that short-circuit network access

    All tests use bun:test. Use createTempDir and mockEnv from helpers.
    Clean up temp dirs in afterEach/afterAll.
  </action>

  <action>
    Create tests/launcher.test.js covering the cross-platform launcher.

    **Testing strategy:** bin/gsd.js is a CLI entry point that spawns claude with args. It cannot be tested by running it directly (would try to launch Claude). Instead:
    1. Extract and test the helper functions by requiring the module in a context where main() does not auto-execute, OR
    2. Test specific behaviors by running with --help or --version flags if supported, OR
    3. Test the logic by examining specific exported/extractable functions

    If gsd.js calls main() at module level (check the file): use approach 2 (CLI subprocess) with controlled args that exit early, or mock spawn/exec to prevent actual Claude launch.

    **Key behaviors to test:**

    Config directory creation:
    - Creates ~/.gsd/ directory if it does not exist
    - Does not error if directory already exists

    Banner display:
    - Outputs GSD banner text to stderr on launch
    - Banner includes version from package.json or VERSION file

    Claude command resolution:
    - Resolves 'claude' command path
    - Handles missing claude command gracefully (error message, not crash)

    Terminal detection:
    - Detects if running in interactive terminal
    - Passes appropriate flags based on terminal capabilities

    Argument forwarding:
    - Forwards extra arguments to claude command
    - Preserves argument order

    **Mocking approach:**
    - Use execSync to run gsd.js with controlled env
    - Set PATH to exclude claude binary (tests command-not-found path)
    - Set HOME/USERPROFILE to temp dir (tests config dir creation)
    - Mock spawn by intercepting with a wrapper script that records args and exits
    - Use timeout to prevent hanging if Claude somehow gets launched

    All tests use bun:test. Clean up temp dirs in afterEach.
  </action>
