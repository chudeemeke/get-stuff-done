## v1 Requirements

Requirements for v0.2.0 Hardening & Upstream Sync milestone. Each maps to roadmap phases.

### Security Hardening

- [x] **SEC-01**: Security review checkpoint in upstream sync workflow blocks cherry-pick until user reviews diff and approves -- Phase 7, wired in Phase 14
- [x] **SEC-02**: Input validation rejects malformed git commit SHAs, branch names with shell metacharacters, and config paths with traversal patterns -- Phase 7, wired in Phase 14 via requireValid bridge
- [x] **SEC-03**: All shell commands use parameterized arguments (no string concatenation), eval() is prohibited -- Phase 7
- [x] **SEC-04**: ESLint with eslint-plugin-security configured and passes on all hooks and installer code -- Phase 7
- [x] **SEC-05**: Publish is blocked if git diff --check detects conflict markers in tracked files -- Phase 7
- [x] **SEC-06**: JSON5 config files are re-validated with AJV after upstream sync cherry-picks are applied -- Phase 14 (validate-configs.js in prepublishOnly)

### Cross-Platform Support

- [x] **PLAT-01**: All path operations use cross-platform library (pathe) instead of string concatenation -- Phase 9
- [x] **PLAT-02**: Platform detection identifies OS (win32/darwin/linux), shell (bash/zsh/PowerShell), and environment (WSL, MINGW) with appropriate fallbacks -- Phase 9
- [x] **PLAT-03**: Filesystem operations validate write permissions before attempting modifications -- Phase 9
- [x] **PLAT-04**: Hook generation produces shell-appropriate syntax for bash, zsh, and PowerShell -- Phase 9
- [x] **PLAT-05**: Installer uses junctions on Windows (no admin) and symlinks on Unix, with automatic fallback to copy mode on permission failure -- Phase 9
- [x] **PLAT-06**: Install, config loading, statusline display, and hook execution verified working on macOS and Linux -- Phase 9, CI matrix in Phase 11, coverage in Phase 16

### Upstream Sync Improvements

- [x] **SYNC-01**: Deferred to v2 scope -- colorized diff preview before cherry-picks
- [x] **SYNC-02**: Deferred to v2 scope -- state snapshots and rollback
- [x] **SYNC-03**: Deferred to v2 scope -- auto-update check with severity indicators
- [x] **SYNC-04**: Deferred to v2 scope -- --dry-run mode for sync operations
- [x] **SYNC-05**: Latest upstream changes are pulled and integrated using the sync workflow -- Phase 8 (72 commits)
- [x] **SYNC-06**: Workflow improvements are implemented based on experience from actual sync execution -- Phase 8

### Claude Code Capability Adoption

- [x] **CLAUDE-01**: GSD agent definitions include memory frontmatter with appropriate scope (user/project/local) -- Phase 10
- [x] **CLAUDE-02**: Agent instructions include extended thinking hints with effort parameter guidance for complex operations -- Phase 10
- [x] **CLAUDE-03**: Agent teams patterns are explored and documented for parallel agent orchestration -- Phase 10, wired in Phase 17
- [x] **CLAUDE-04**: Excluded per CONTEXT.md -- fast mode integration (marginal benefit)
- [x] **CLAUDE-05**: Excluded per CONTEXT.md -- bash-to-Claude-tools migration (bash has valid advantages)

### CI/CD

- [x] **CI-01**: GitHub Actions workflow runs cross-platform matrix testing on macOS, Linux, and Windows -- Phase 11

## Out of Scope

| Feature | Reason |
|---------|--------|
| Rewriting GSD core methodology | Fork adaptation, not a rewrite |
| Removing .upstream/ directory | Needed for diffing/comparison |
| TypeScript migration | Adds build complexity for marginal benefit in this codebase |
| Heavy testing frameworks | bun:test sufficient; Jest/Mocha overkill |
| Docker/containerization | GSD runs natively in Claude Code CLI |
| Server/HTTP layer | GSD is CLI-based, not server-based |
| Database/persistent storage | Filesystem (JSON config, git) is sufficient |
| Auto-applying upstream updates | Destroys user control; always require confirmation |
| Storing secrets in config | Security vulnerability for git-based config |
