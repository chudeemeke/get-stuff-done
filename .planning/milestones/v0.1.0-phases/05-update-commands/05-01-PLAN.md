3. **Cache handling** - After successful update:
   - Clear update check cache: `rm -f ~/.claude/cache/gsd-update-check.json`
   - Update .planning/sync/cache.json with new version and date using jq

4. **VERSION file location** - Currently reads from `~/.claude/get-stuff-done/VERSION` which is correct

5. **Add cache update step** - After successful install, update .planning/sync/cache.json using jq for reliable nested JSON updates:
```bash
# Update sync cache with installed version (using jq for nested structure)
if [ -d ".planning/sync" ] && [ -f ".planning/sync/cache.json" ]; then
  jq --arg v "${NEW_VERSION}" --arg d "$(date -Iseconds)" \
    '.last_update.version = $v | .last_update.date = $d' \
    .planning/sync/cache.json > /tmp/gsd-cache.tmp && \
    mv /tmp/gsd-cache.tmp .planning/sync/cache.json
fi
```

IMPORTANT: Use jq (not sed) for cache updates because cache.json has nested structure (last_update.version, last_update.date). sed patterns would incorrectly match any "version" or "date" field in the JSON.

6. **Preserve existing flow** - The current process flow is good:
   - Get installed version
   - Check latest version
   - Compare versions
   - Show changes and confirm
   - Run update
   - Display result

7. **Branding consistency** - Ensure all output uses "GSD" branding, not "Get Shit Done"

Keep the existing skill structure (frontmatter, objective, process steps, success_criteria). Only modify URLs and add cache update logic.
