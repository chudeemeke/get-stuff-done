<objective>
Bundle gsd-tools.js via esbuild so src/validation is inlined, fixing MODULE_NOT_FOUND when installed via copy-mode to ~/.claude/get-stuff-done/bin/.

Purpose: Phase 14 added `require('../../src/validation')` to gsd-tools.js. This works at source root but breaks after copy-mode install because ~/.claude/src/validation/ does not exist. This is the same class of bug Phase 13 fixed for hooks -- same fix pattern applies.

Key decision: Consolidate `scripts/build-hooks.js` into a unified `scripts/build.js` via `git mv`, then add gsd-tools.js as a second build target. This is a user-locked decision from 15-CONTEXT.md -- do NOT create a separate `scripts/build-tools.js`.

Output: Unified build script, self-contained bundled gsd-tools.js in get-stuff-done/bin/dist/, installer wired to use it, dist regression tests as permanent guard.
</objective>
