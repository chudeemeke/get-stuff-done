## Implementation Decisions

### Re-verification Methodology (Criterion #4)

- **D-01:** Verification depth for the 20 D-XX/A-XX items NOT explicitly named in success criterion #4: **per-decision matrix, semantic check** (cited symbol/API/line behavior re-verified, not just file existence). The 2 explicitly named (D-09 vs upstream #2499; 999.1 vs upstream #2487) get the same treatment.

- **D-02:** Matrix ordering: **by upstream-touch surface**. Decisions citing upstream files (`ci.yml` lines, `scripts/check-overrides.js`, `tests/helpers/test-timeouts.js`) ranked first; fork-only file citations last. They can't drift from upstream so verification load is minimal.

- **D-03:** Amendments captured as **A-07+** in Phase 41 CONTEXT.md's existing Amendments Log (chronological log; original D-XX content amended in-place). Maintains continuity with the locked-CONTEXT convention.

- **D-04:** Verification owner: **interactive Claude in plan/execute (NOT subagent)**. User wants to be in the loop on verification thinking, not just sign-off. **Implication for the planner:** tasks must be sized for interactive execution with user-attention budget per row. No batch verifier-agent step.

- **D-05:** Drift signal threshold: **cited symbol/API/line behavior changed**. Whitespace/comment-only delta near the citation = unchanged. Behavior delta = drifted. Mechanical signal (file moved/renamed but behavior preserved) = drifted with auto-amendment proposal.

- **D-06:** Amendment sign-off gate: **user reviews each amendment** before commit. Honors locked-CONTEXT principle from the v1.2.0 milestone start.

### Override Drift + #1859 Cleanup (Criterion #2 + #3)

- **D-07:** Default response on SHA-256 mismatch (override snapshot vs current upstream): **human review every drift**. AST-diff is Phase 43 territory (UPGRADE-04); until then each drift gets human eyes per skin-fork principle. No regex/heuristic auto-classification.

- **D-08:** Default action on overlay/upstream collision: **compare semantics, then decide**. Equivalent → remove overlay, use upstream. Meaningfully divergent → move to overrides/ with REASON.md + SHA snapshot.

- **D-09:** Handling for PR #1859 (review.md per-CLI model selection, merged upstream 2026-04-10 via the now-current 1.38.2): **audit + remove if equivalent**. Diff fork's `overlay/review.md` (and config.cjs deltas) against upstream's merged version; remove if equivalent, otherwise move to `overrides/` with REASON.md citing the divergence.

### Verification Artifact Shape (Criterion #5)

- **D-10:** `40.5-VERIFICATION.md` uses **standard `gsd-verifier` output** — no custom shape. GSD framework owns the template via the verifier agent.

- **D-11:** `40.5-DECISION-VERIFY.md` (separate evidence file): 22-row matrix. Columns: **ID | Cited target | Upstream commit/SHA | Status | Evidence | Amendment ref**. Status enum: `unchanged | drifted | amended`. Cited inline by VERIFICATION.md as criterion #4 evidence.

- **D-12:** `40.5-DRIFT-LOG.md` (separate evidence file): per-file SHA mismatch + semantic compare + refresh-or-amend decision. Cited inline by VERIFICATION.md as criterion #2 evidence.

- **D-13:** `40.5-COLLISIONS.md` (separate evidence file): per-overlay collision + keep/move/remove decision (includes #1859 audit per D-09). Cited inline by VERIFICATION.md as criterion #3 evidence.

- **D-14:** Refreshed override snapshots: **committed inline** with the verification work; no separate aggregator file. Each override's REASON.md gets its `Snapshot:` field updated atomically with the verification commit.

- **D-15:** `40.5-README.md`: short index of phase-dir contents and why each exists. Self-documenting; zero framework dependency. Combines with VERIFICATION.md forward-citations to give two-layer orphan-prevention.
