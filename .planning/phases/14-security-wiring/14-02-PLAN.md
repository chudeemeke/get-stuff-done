Wire validateConfigPath into ConfigLoader.js to validate the GSD_CONFIG_PATH environment variable. This is a defense-in-depth measure -- the env var takes an arbitrary path from the environment and uses it to read a config file.

**Step 1: Add import in ConfigLoader.js (after existing requires):**
```javascript
const { validateConfigPath } = require('../validation');
```

**Step 2: Validate GSD_CONFIG_PATH in getConfigPath():**

Currently (line 11-15):
```javascript
function getConfigPath() {
  if (process.env.GSD_CONFIG_PATH) {
    return process.env.GSD_CONFIG_PATH;
  }
  return path.join(os.homedir(), '.gsd', 'config.json');
}
```

Change to:
```javascript
function getConfigPath() {
  if (process.env.GSD_CONFIG_PATH) {
    const result = validateConfigPath(process.env.GSD_CONFIG_PATH, os.homedir());
    if (!result.ok) {
      throw new Error(`GSD_CONFIG_PATH validation failed: ${result.error}`);
    }
    return result.value;
  }
  return path.join(os.homedir(), '.gsd', 'config.json');
}
```

The base directory for validation is `os.homedir()` -- config files should be within the user's home directory. This prevents path traversal attacks via the env var (e.g., GSD_CONFIG_PATH=/etc/shadow).

Note: ConfigLoader throws on errors (it already throws for parse errors at line 128). This is consistent with the module's existing error handling pattern. The validation module returns Result type; ConfigLoader converts to throw (same as how gsd-tools.js uses requireValid to convert to process.exit).

**Step 3: Update tests in tests/config.test.js:**

The existing test `'uses GSD_CONFIG_PATH env var when set'` (line 36) sets GSD_CONFIG_PATH to `/custom/path/config.json`. This will now fail validation because `/custom/path/` is likely not within `os.homedir()`. Update:

test('uses GSD_CONFIG_PATH env var when set', () => {

2. Add a new test for GSD_CONFIG_PATH path traversal rejection:
```javascript

  restoreEnv = mockEnv({ GSD_CONFIG_PATH: '/etc/shadow' });
  expect(() => getConfigPath()).toThrow('GSD_CONFIG_PATH validation failed');
});
```

3. Update any other tests that set GSD_CONFIG_PATH to ensure the paths are within os.homedir(). Check tests at lines 101, 116, 135, 146, 154 -- these use temp directories which may or may not be within homedir. If they use createTempDir(), the temp dir is likely in os.tmpdir() which is NOT within homedir. You may need to either:
   - Create temp dirs within homedir for these tests, OR
   - Use a broader base directory for validation in tests

The pragmatic approach: for tests that need to write actual config files, create the temp dir as a subdirectory of os.homedir() (e.g., `path.join(os.homedir(), '.gsd-test-' + Date.now())`), or mock validateConfigPath to always return ok for those specific tests.

Actually, re-examining: the tests at lines 101, 116, 135, etc. set GSD_CONFIG_PATH to paths within a temp dir (created by createTempDir). Since temp dirs are typically at /tmp/ or C:\Users\X\AppData\Local\Temp, they're NOT within homedir. The cleanest solution: in getConfigPath(), use the PARENT directory of the config path as the base, not homedir. But that defeats the purpose.

Better approach: use `os.tmpdir()` as an ADDITIONAL allowed base. Or better yet: the base directory for GSD_CONFIG_PATH validation should be broad enough to cover legitimate use cases (homedir for normal use, tmpdir for testing). Use an array of allowed bases:

```javascript
function getConfigPath() {
  if (process.env.GSD_CONFIG_PATH) {
    // Validate path is within user-accessible directories
    const allowedBases = [os.homedir(), os.tmpdir()];
    let validated = false;
    for (const base of allowedBases) {
      const result = validateConfigPath(process.env.GSD_CONFIG_PATH, base);
      if (result.ok) {
        validated = true;
        return result.value;
      }
    }
    throw new Error(`GSD_CONFIG_PATH validation failed: path must be within home or temp directory`);
  }
  return path.join(os.homedir(), '.gsd', 'config.json');
}
```

This allows homedir (production) and tmpdir (testing) while still rejecting /etc/shadow and other system paths.

Update tests to verify: valid homedir path works, valid tmpdir path works, /etc/shadow rejected.
