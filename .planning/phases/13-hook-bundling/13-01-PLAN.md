<task type="auto">
  <name>Task 1: Replace plain-copy build-hooks.js with esbuild bundling</name>
  <files>scripts/build-hooks.js</files>
  <action>
Rewrite `scripts/build-hooks.js` to use esbuild's programmatic API instead of `fs.copyFileSync`.

**Implementation details:**

1. Replace the `fs.copyFileSync` loop with `esbuild.buildSync` calls for each hook.

2. Use these exact esbuild options for ALL three hooks:
```javascript
const esbuild = require('esbuild');

esbuild.buildSync({
  entryPoints: [path.join(HOOKS_DIR, hookFile)],
  bundle: true,           // Inline all require()d modules
  platform: 'node',       // Target Node.js (CJS output, correct pathe resolution)
  target: 'node20',       // Match engines.node in package.json
  outfile: path.join(DIST_DIR, hookFile),
  // NO 'external' -- everything must be inlined for copy-mode install
  // NO 'format: esm' -- platform:'node' defaults to CJS which is correct
  // NO 'minify' -- keep readable for debugging installed hooks
  // esbuild auto-preserves #!/usr/bin/env node shebang
});
```

3. Bundle all three hooks: `pre-compact.js`, `gsd-statusline.js`, `gsd-check-update.js`. The last one has no src/ imports but bundle it for consistency (esbuild strips comments, producing slightly smaller output).

4. Keep the `HOOKS_DIR` and `DIST_DIR` constants. Keep the `mkdir` for dist/. Remove the `HOOKS_TO_COPY` array and rename it to `HOOKS_TO_BUNDLE` or similar.

5. Update the comment on what was line 14 (`// Hooks to copy (pure Node.js, no bundling needed)`) -- this comment is now wrong. Update to reflect bundling.

6. Add console.log output showing hook name and bundle result (e.g., "Bundling pre-compact.js... done (23KB)"). Include file size in output for visibility.

7. Keep the eslint-disable comment at top (security/detect-non-literal-fs-filename still applies for fs.mkdirSync on DIST_DIR).

8. After writing, run the build: `bun run build:hooks` and verify all three dist/ files are created and are larger than 1KB (confirming actual bundling, not empty output).

**What NOT to do:**
- Do NOT use esbuild CLI via child_process.spawn (use programmatic API)
- Do NOT add `external` for any package (ajv, json5, pathe must all be inlined)
- Do NOT add `format: 'esm'` (breaks pathe CJS resolution)
- Do NOT add `minify: true` (readability for debugging)
- Do NOT use `metafile: true` (out of scope)
