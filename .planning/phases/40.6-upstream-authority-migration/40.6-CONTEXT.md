# Phase 40.6 Context: Upstream Authority Migration

**Inserted:** 2026-06-22  
**Milestone:** v1.2.0 Ship-Ready Hardening  
**Requirement:** UPGRADE-11  
**Status:** Planned, pending review before implementation

## Objective

Move `@chude/get-stuff-done` from the legacy upstream authority
`get-shit-done-cc` / `gsd-build/get-shit-done` to the active Open GSD authority
`@opengsd/gsd-core` / `open-gsd/gsd-core`, while preserving:

- The overlay architecture.
- Fork identity and branding.
- Exact reviewed upstream version pins.
- Override staleness checks.
- Phase 41 ship-ready hardening intent.

This is a platform-safety phase. All projects use GSD, so the blast radius is every
project that depends on the user's GSD workflow.

## Why This Phase Exists

Phase 40.5 started as a normal upstream currency bump. That assumption is no longer
good enough. Live upstream review on 2026-06-22 showed:

- The old package `get-shit-done-cc` still publishes versions, but remains tied to
  the legacy maintainer/account.
- The old repository now redirects socially and procedurally toward Open GSD.
- Open GSD is the active continuation, but its package name and bin/source layout
  differ from the old package.
- Phase 40.5 Wave 5 still contains instructions to file work against the legacy
  upstream. That is now unsafe as the default path.

Phase 40.6 makes the authority change explicit before Phase 41 adds more hardening on
top of the wrong foundation.

## Non-Goals

- No rewrite of the GSD methodology.
- No migration to a separate "GSD-2" architecture.
- No auto-tracking `latest`, `next`, canary, or dev tags.
- No patching global installed GSD while investigating.
- No silent deletion of historical references in archived milestones.

## Decisions

### D-01: Open GSD core is the target authority

The target authority is `@opengsd/gsd-core` / `open-gsd/gsd-core`, initially pinned to
stable `1.5.0` based on 2026-06-22 evidence. The legacy package is now migration
evidence, not the active upstream target.

### D-02: Exact pinning still governs upstream updates

The migration target is not `@opengsd/gsd-core@latest`. The first implementation
must pin an explicitly reviewed stable version. Future Open GSD bumps remain
deliberate version changes.

### D-03: Package layout must be discovered before implementation

`@opengsd/gsd-core` is not assumed to be a drop-in replacement. The known bin shape
differs from the legacy package; source root and file layout must be inspected in an
isolated temp install before touching production tooling.

### D-04: Centralize upstream identity

Current tooling hardcodes `get-shit-done-cc` in multiple scripts. Phase 40.6 must
centralize active upstream identity in a manifest/helper before or during the code
migration, so future authority changes are localized.

### D-05: Preserve overlay architecture

The fork continues to consume upstream as a build-time dependency, compose upstream
plus overlay/overrides into `dist/`, and publish `@chude/get-stuff-done` as a
self-contained package.

### D-06: Legacy filing path is suspended

Phase 40.5 Wave 5's legacy upstream filing path is suspended. Phase 40.6 decides
whether any old `gsd-build/get-shit-done` filing remains useful after the authority
migration.

### D-07: Review before implementation

Because this phase changes the supply-chain root for the GSD overlay, the four plans
must be reviewed before code migration starts.

### D-08: Authority manifest scope is build-time unless explicitly packaged

`.planning/upstream-authority.json` is the maintainer/build-time source of truth. No
published runtime path may depend on `.planning/` existing. If package-shipped code
needs authority metadata, the migration must generate or include a package artifact
explicitly and verify it through pack/install testing.

### D-09: Package integrity evidence is required before layout trust

The Open GSD package layout spike must record npm dist integrity, shasum, tarball URL,
license, package scripts, and release/tag correlation before the package is trusted as
the new authority.

### D-10: Phase 40.6 cannot close without package-artifact verification

Source-level compose/tests are insufficient. Phase 40.6 completion requires a packed
artifact check and scratch consumer install or an explicit environment blocker that
prevents completion.

## Exit Criteria

Phase 40.6 is complete only when:

1. Authority decision is recorded in an ADR and machine-readable manifest/helper.
2. Open GSD package layout is inspected in isolation and documented.
3. Tooling reads the new authority without scattered active hardcodes.
4. Compose, override check, boundary check, preview-update, tests, and legacy-reference
   audit pass against Open GSD.
5. Package dry-run/pack and scratch consumer install verification pass against the
   composed artifact.
6. Phase 41 readiness is reset against the Open GSD authority.
