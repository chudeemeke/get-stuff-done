## Version-Refresh Primitive (referenced by Tasks 04-02..04-05)

For external GitHub Actions and CLI tools, the executor needs a version-refresh
verification command. Use these in priority order:

1. **Primary (GitHub Actions / repos):** `gh release list -R <owner>/<repo> --limit 5 | head -1` — captures the latest tag. Requires `gh` CLI authenticated.
2. **Fallback (no gh / not authenticated):** `curl -s https://api.github.com/repos/<owner>/<repo>/releases/latest | jq -r .tag_name` — works without gh auth (GitHub anonymous API rate limit applies but is generous for this volume).
3. **npm-published packages:** `npm view <package> version` — different ecosystem from GitHub releases. Use this for packages like `markdownlint-cli2`.
4. **No-tooling fallback:** mark `unchanged-presumed-no-tooling` in the External-action refresh table; add a one-line callout to VERIFICATION.md criterion #4 evidence noting which tools were unavailable in the executor's environment.

Each row in the External-action refresh table MUST cite the version-refresh command actually run (or `unchanged-presumed-no-tooling` if neither tool was available).
