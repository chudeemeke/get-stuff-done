  <name>Task 2: GREEN -- Implement preview-update script</name>

  <action>
Create scripts/preview-update.js (~250-350 lines). This is a read-only CLI script that outputs a structured report. It modifies nothing.

Implementation structure:

**getVersionDelta():**
- Read pinned version from package.json devDependencies['get-shit-done-cc']
- Query latest version via `npm view get-shit-done-cc version` (execSync, encoding: 'utf-8', trim)
- Return { pinned, latest, hasUpdate: pinned !== latest }
- Handle npm view failure gracefully (network error, package not found)

**runPreviewScan(pinnedVersion, latestVersion):**
- Import runSupplyChainChecks from overlay/lib/sync.cjs
- For version-level scanning (not git-commit-level), adapt the call:
  - Generate a synthetic diff context focused on file-path-based vectors (execution-path changes, prompt-integrity)
  - Use `npm pack get-shit-done-cc@<version> --pack-destination <tmpDir>` to get tarball, extract file lists for comparison. If too slow, fall back to file-list-only analysis
  - Skip author-anomaly check (irrelevant for npm package version comparison)
- Return array of findings: Array<{ check, severity, triggered, evidence }>
- Flag critical findings prominently in output (per CONTEXT.md) but do NOT block

**getOverrideImpact():**
- Import checkOverrides from scripts/check-overrides.js
- Run checkOverrides() with default paths (overrides/ and node_modules/get-shit-done-cc/)
- Return the result: { ok, overrides, summary }
- If zero overrides (day-one state): return early with clean status

**generateReport(versionDelta, scanFindings, overrideImpact):**
- Produce structured text output to stdout with sections:
  1. **Version Delta**: Current pinned -> Latest available, files added/removed/changed count
  2. **Supply Chain Scan**: List findings by severity. Critical findings get prominent markers. No findings = "Clean scan"
  3. **Override Impact**: List stale overrides needing review. No overrides = "No overrides to review"
  4. **Next Steps**: If update available:
     - "To apply this update:"
     - "1. Edit package.json: set get-shit-done-cc to <latest>"
     - "2. Run: bun install"
     - "3. Run: bun run compose"
     - "4. Run tests: bun test"
  5. **Rollback** (UPD-04): "To rollback: pin the previous version (<pinned>) in package.json, run bun install, run bun run compose"
- If no update available: "Already up to date (pinned: <version>)"

**CLI entry point (when run directly):**
- Print header with script name
- Call getVersionDelta()
- If no update: print "up to date", exit 0
- Call runPreviewScan()
- Call getOverrideImpact()
- Call generateReport()
- Print report to stdout
- Exit 0 (read-only, always succeeds unless npm view fails)

**Module exports (for testing):**
- Export: getVersionDelta, runPreviewScan, getOverrideImpact, generateReport

Add `"preview-update": "node scripts/preview-update.js"` to package.json scripts section.
