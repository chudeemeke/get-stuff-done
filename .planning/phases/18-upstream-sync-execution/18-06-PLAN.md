  <name>Task 1: Add --help flag handling to gsd-tools CLI router</name>
  <files>get-stuff-done/bin/gsd-tools.cjs</files>
  <action>
In `get-stuff-done/bin/gsd-tools.cjs`, the `main()` function at line 140 handles `!command` (no args) by showing usage via `error()`, but `--help` is treated as a command name and falls to the `default:` case which errors with "Unknown command: --help".

Fix: After the `--raw` flag extraction (line 142-144) and before the `!command` check (line 149), add detection for `--help` and `-h` flags. When detected, print the same usage string currently in the `!command` check to stdout (NOT stderr via error()), then exit 0 (success, not failure).

Specifically:
1. After line 144 (`if (rawIndex !== -1) args.splice(rawIndex, 1);`), add:
   ```
   if (args.includes('--help') || args.includes('-h')) {
     process.stdout.write('Usage: gsd-tools <command> [args] [--raw]\n\nCommands:\n  state load|update|get|patch    Manage STATE.md\n  resolve-model <agent-type>     Get model for agent\n  find-phase <phase>             Find phase directory\n  commit <message> [--files ..]  Commit planning docs\n  verify-summary <path>          Verify SUMMARY.md\n  generate-slug <text>           Text to URL slug\n  current-timestamp [format]     ISO timestamp\n  list-todos [area]              Enumerate pending todos\n  verify-path-exists <path>      Check file existence\n  config-ensure-section          Init config.json\n  history-digest                 Aggregate SUMMARYs\n  summary-extract <path>         Extract SUMMARY data\n  state-snapshot                 Parse STATE.md\n  phase-plan-index <phase>       Index plans with waves\n  phase next-decimal|add|insert|remove|complete  Phase ops\n  roadmap get-phase|analyze|update-plan-progress  Roadmap ops\n  requirements mark-complete <ids>  Mark requirements done\n  milestone complete <version>   Archive milestone\n  validate consistency|health    Check integrity\n  progress [json|table|bar]      Render progress\n  init <workflow> <phase>        Init workflow context\n  frontmatter validate|extract   Frontmatter ops\n  template list|get|apply        Template ops\n  websearch <query>              Web search via Brave API\n\nFlags:\n  --raw    Output raw values without JSON wrapping\n  --help   Show this help message\n');
     process.exit(0);
   }
   ```
2. Also add `'help'` as a case in the switch statement (before `default:`) that does the same stdout write + exit 0. This way both `gsd-tools --help` and `gsd-tools help` work.

Key points:
- Use `process.stdout.write` (NOT the `error()` function which writes to stderr and exits with code 1)
- Exit with code 0 (success) for help display
- The usage text should list all commands from the comment block at top of file (lines 12-90)
