  <name>Task 1: Audit and fix gsd-tools.js for cross-platform compatibility</name>
  <files>get-stuff-done/bin/gsd-tools.js</files>
  <action>
Perform a systematic audit of gsd-tools.js (1,878 lines) for cross-platform issues. Known issues from research:

1. **Line ~1240: Unix `find` command** - `cmdInitNewProject` uses `execSync('find . -maxdepth 3 ...')` which fails on Windows (no `find` equivalent in cmd.exe/PowerShell). Replace with Node.js `fs.readdirSync` recursive walk:
   - Create a helper function `findCodeFiles(cwd, maxDepth)` that walks directories up to maxDepth
   - Filter for extensions: .ts, .js, .py, .go, .rs, .swift, .java
   - Exclude node_modules and .git directories
   - Return first 5 matches (matching `head -5` behavior)
   - Wrap in try/catch (same error handling as current)

2. **Lines ~109, ~125: Shell command construction** - `execGit` and `isGitIgnored` construct shell commands via string concatenation. Verify these work on Windows:
   - `git check-ignore -q` works on Windows Git (no change needed, git is git everywhere)
   - Shell quoting uses single quotes which don't work in cmd.exe. However, `execSync` with default options uses `/bin/sh` on Unix and `cmd.exe` on Windows. On Windows, single-quote wrapping breaks. Fix by using `execFileSync` with array args instead of `execSync` with string commands for both `execGit` and `isGitIgnored`.
   - `execFileSync('git', ['check-ignore', '-q', '--', targetPath], { cwd, stdio: 'pipe' })` -- no shell needed
   - `execFileSync('git', args, { cwd, stdio: 'pipe', encoding: 'utf-8' })` -- direct args, no quoting

3. **Full file scan** for any other cross-platform issues:
   - Search for hardcoded `/` path separators in file operations (should use path.join)
   - Search for `process.env.HOME` (should use `os.homedir()`)
   - Search for Unix-specific commands (grep, sed, awk, cat, rm, chmod, mkdir -p)
   - Search for `\n` line ending assumptions (should be `os.EOL` or explicit `\n` for git/config)
   - Note: Use `\n` explicitly for config/markdown files (not os.EOL) -- git normalizes line endings

4. **Preserve all existing behavior** - This is a bug-fix audit, not a refactor. Only change what breaks on Windows.

Important: gsd-tools.js is upstream code integrated in Phase 8. Make minimal, targeted changes. Document each change with a comment explaining why (e.g., `// Cross-platform: replaced Unix find with Node.js walk`).
  </action>

    - Search for `execSync.*find ` in gsd-tools.js returns zero matches (Unix find removed)

<task type="auto">
  <name>Task 2: Harden installer with permission validation and platform detection</name>
  <files>bin/install.js</files>
  <action>
Improve the installer (bin/install.js) with better cross-platform handling:

1. **Permission validation before symlink/junction/copy:**
   - Before attempting symlink creation, test write permissions on the target directory:
     ```javascript
     function canWrite(dir) {
       try {
         const testFile = path.join(dir, '.gsd-write-test-' + Date.now());
         fs.writeFileSync(testFile, '');
         fs.unlinkSync(testFile);
         return true;
       } catch { return false; }
     }
     ```
   - If write permission fails, display clear error with platform-specific advice (e.g., "Run as administrator" on Windows, "Check directory permissions" on Unix)

2. **Improve createSymlink fallback chain:**
   - Current implementation (line ~228) already handles junction fallback on Windows
   - Add: try symlink -> catch EPERM -> try junction -> catch -> copy with warning
   - On EPERM from symlink, inform user: "Symlinks require Developer Mode on Windows. Using junction instead."
   - On junction failure, inform user: "Junction creation failed. Falling back to file copy. Note: changes to source won't auto-propagate; reinstall after updates."
   - Record the actual method used (symlink/junction/copy) in install metadata

3. **Enrich install metadata in .install-meta.json:**
   - Add platform detection results: os, shell, environment (isMingw, etc.)
   - Add node version: process.version
   - Add install method with detail: { method: 'symlink'|'junction'|'copy', reason: 'default'|'eperm_fallback'|'write_error_fallback' }
   - Import and use `src/platform/detect` for platform info (note: at install time the platform module may not be in the target yet, so detect inline or require from source)

4. **Hook command registration:**
   - When registering hooks in settings.json, register `pre-compact.js` (the new Node.js hook) instead of `pre-compact.sh`
   - Add PreCompact hook entry alongside existing SessionStart hook entry
   - Use `buildHookCommand(claudeDir, 'pre-compact.js')` for the command string

5. **Update Node.js version requirement:**
   - Change `engines.node` in package.json from `>=16.7.0` to `>=20.0.0` (per CONTEXT.md: minimum Node.js 20 LTS)
   - Add a runtime check at the top of install.js: if Node.js < 20, warn and continue (don't block)

Important: Keep all existing functionality intact. These are additions/improvements, not rewrites.
  </action>
