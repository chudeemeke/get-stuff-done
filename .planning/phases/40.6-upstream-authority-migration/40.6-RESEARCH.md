## Public Governance Signal

Open GSD discussion #109 (`https://github.com/open-gsd/gsd-core/discussions/109`)
states that `open-gsd/gsd-core` is a community-maintained continuation of the work
originally published at `gsd-build/get-shit-done`. It also separates what the Open GSD
maintainer says they can confirm from what they cannot confirm.

Use this evidence carefully:

- Treat the old upstream as a supply-chain and governance risk.
- Do not infer personal safety, account control, or legal culpability beyond public
  evidence.
- Do not repeat unverified social-media claims as project facts.
- The project-relevant conclusion is enough: active authority should move to Open GSD.

## Migration Implications

1. The new Open GSD package has a scoped name and different bin keys.
2. The source root appears to be nested under `gsd-core/`, but this must be verified by
   extracting/installing the package in a temp directory.
3. `@opengsd/gsd-core` has no `gsd-sdk` bin in the observed npm metadata; any tooling
   assuming that bin must be retested.
4. `preview-update.js` must query the new package and produce Open GSD-aware upgrade
   instructions.
5. Override source snapshot paths must resolve against the new package layout.
6. Branding rules must not corrupt internal path names.

## Recommendation

Proceed with Phase 40.6 before Phase 41:

- Pin `@opengsd/gsd-core@1.5.0` initially.
- Create an authority ADR and manifest/helper.
- Run an isolated package-layout spike.
- Implement identity abstraction and migrate tooling.
- Verify no active tooling relies on legacy `get-shit-done-cc`.
