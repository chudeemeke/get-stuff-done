# Milestone v0.2.0: Hardening & Upstream Sync

**Status:** SHIPPED 2026-02-21
**Phases:** 7-17
**Total Plans:** 32

## Overview

v0.2.0 establishes production-grade quality through security hardening, upstream sync, cross-platform support, and modern Claude Code integration. The roadmap follows a security-first dependency chain: audit and harden the codebase, sync upstream changes (using Phase 7 security tools for safe execution), enable cross-platform support on the synced codebase, adopt Claude Code 2026 capabilities, and optionally add CI/CD automation. This milestone transforms GSD from a working fork into a reliable, maintainable tool that safely tracks upstream while preserving distinct identity.

## Milestone Summary

**Key Decisions:**

- Security-first phase ordering (audit before features)
- Upstream features take priority over fork implementations on conflict
- Sync to latest upstream at time of execution (not pinned to specific version)
- pathe library for cross-platform path normalization
- Singleton pattern for platform/terminal detection
- bun:test over node:test for performance
- esbuild bundling for hooks and gsd-tools.js (copy-mode install support)
- Result type pattern for validation (no exceptions thrown)
- Agent teams disabled by default (experimental), two-gate safety (config + env flag)
- 144 total decisions tracked in STATE.md

**Issues Resolved:**

- Hook MODULE_NOT_FOUND on copy-mode install (Phase 13: esbuild bundling)
- Validation module orphaned from production code (Phase 14: requireValid bridge)
- Config re-validation missing after upstream sync (Phase 14: validate-configs.js)
- gsd-tools.js MODULE_NOT_FOUND post-install (Phase 15: esbuild bundling)
- Platform coverage below 95% threshold (Phase 16: internal helper exports)
- Dead code src/platform/index.js (Phase 16: deleted)
- Agent teams not wired into workflows (Phase 17: config-driven routing)

**Issues Deferred:**

- SYNC-01: Colorized diff preview before cherry-picks (v2 scope)
- SYNC-02: State snapshots and rollback (v2 scope)
- SYNC-03: Auto-update severity indicators (v2 scope)
- SYNC-04: --dry-run mode for sync operations (v2 scope)

**Technical Debt:**

- 95 ESLint warnings in gsd-tools.js (upstream code, reviewed and non-exploitable)
- Inline platform detection in install.js instead of src/platform module (works correctly)
