## Gaps

### Critical (blocks ship)

**GAP-1: Installed hooks crash on copy-mode install (Phases 9/11)**

Hooks `pre-compact.js` and `gsd-statusline.js` use `require('../src/...')` relative imports. When the installer copies hooks from `hooks/dist/` to `~/.claude/hooks/`, these resolve to `~/.claude/src/` which does not exist. The `build-hooks.js` script performs a plain file copy with no import resolution.

- `pre-compact.js` line 21: `require('../src/platform/paths')` -> MODULE_NOT_FOUND
- `gsd-statusline.js` line 22: `require('../src/config/ConfigLoader')` -> try-catch, silent degrade
- `gsd-statusline.js` line 32: `require('../src/theme')` -> MODULE_NOT_FOUND crash
- `gsd-statusline.js` line 48: `require('../src/platform/terminal')` -> MODULE_NOT_FOUND crash

Impact: Context compaction hook and statusline hook are non-functional for npm consumers using copy-mode install. Dev-mode (`--link`) is unaffected since symlinks preserve relative paths.

Fix options: (a) Bundle hooks with esbuild to inline dependencies, (b) Install src/ alongside hooks, (c) Inline required functions into hook files, (d) Use dynamic path resolution from npm package location.

### Non-Critical (can ship with known issues)

**GAP-2: Phase 7 validation module orphaned (SEC-01, SEC-02)**

`src/validation/index.js` exists with correct implementation but has 0 production callers. Input validation for git SHAs, branch names, and config paths is not enforced at runtime. gsd-tools.js line 990 uses its own inline regex.

Impact: Security validation exists but is documentation-only. Not blocking since GSD operates in a trusted CLI environment, but violates the stated security hardening intent.

**GAP-3: Phase 7 config re-validation missing (SEC-06)**

No executable code implements config re-validation after upstream sync cherry-picks. Step is documented in workflow for Claude agents but not enforced by hooks or scripts.

Impact: If upstream sync modifies config files, they may not be re-validated before publish. prepublishOnly script checks for conflict markers but not schema validity.

**GAP-4: Phase 11 has no formal VERIFICATION.md**

Phase 11 was never verified by gsd-verifier. Phase has UAT results (6/7 pass) and comprehensive summaries, but the formal verification step was skipped.

Impact: Procedural gap only. Phase functionality is confirmed by summaries, test results (355 tests), and UAT.
