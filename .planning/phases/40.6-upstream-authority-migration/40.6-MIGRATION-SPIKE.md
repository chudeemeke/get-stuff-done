behavior change is path mapping inside that package root, especially workflows and
`gsd-tools`.

## Implementation Notes For Plan 03

- Add `scripts/lib/upstream-source.js` to resolve scoped package roots safely.
- Replace hardcoded `DEFAULT_UPSTREAM_DIR = node_modules/get-shit-done-cc`.
- Replace `REQUIRED_UPSTREAM_DIRS = ['agents', 'bin', 'commands', 'get-shit-done', 'hooks', 'scripts']` with Open GSD-aware requirements.
- Replace `PROMPT_PATH_PREFIXES.workflows = 'get-shit-done/workflows/'` with manifest-backed `gsd-core/workflows/`.
- Remove active `gsd-sdk` assumptions; Open GSD core has no `gsd-sdk` bin.
- Use argv-based npm queries in `preview-update.js`; do not shell-string scoped package names.
- Keep `.planning/upstream-authority.json` build-time only unless a package-included generated artifact is explicitly added.

## Risks

- Current package `files` includes `bin` but not `scripts`. `bin/preview-update-cli.js`
  requires `../scripts/preview-update`; this may already be a package artifact issue.
  Plan 04 package smoke must catch this class.
- Open GSD has many more package surfaces than legacy. Boundary checks must distinguish
  active package root changes from historical doc references.
- Lifecycle scripts exist in Open GSD package. Verification must decide deliberately
  whether to ignore or allow scripts in scratch installs.
