    - validateConfig() rejects invalid types (string where number expected)

    - Accepts full 40-char hex SHA
    - Accepts short 7-char hex SHA
    - Accepts mixed-case hex (case insensitive)
    - Rejects non-hex characters (g, z, special chars)
    - Rejects too-short SHA (6 chars)
    - Rejects too-long SHA (41 chars)
    - Rejects empty string
    - Rejects non-string input (number, null, undefined)

    **validateBranchName tests:**
    - Accepts standard branch names (main, feature/add-login, fix-123)
    - Accepts names with dots, hyphens, underscores, slashes
    - Rejects shell metacharacters (;, |, &, $, backtick, etc.)

    - Rejects names with .. (git traversal)
    - Rejects empty string
    - Rejects non-string input

    - Accepts paths within allowed base directory
    - Rejects path traversal (../../etc/passwd)
    - Rejects URL-encoded traversal (%2e%2e)
    - Handles Windows backslash paths
    - Rejects paths outside base directory

    All tests use bun:test (describe, test, expect). Use expect().toThrow() for error cases.

    - Test clearCache behavior (functional, not platform-dependent)
    - Test that return shape is correct regardless of platform
    - Test shell detection logic by mocking environment variables (SHELL, ComSpec, PSModulePath) after clearCache
