**Step 1: Merge get-shit-done/ into get-stuff-done/**

After cherry-picking upstream commits, a `get-shit-done/` directory likely exists alongside the fork's `get-stuff-done/`. Merge them:

a. Compare the two directories:
   ```bash
   diff -rq get-shit-done/ get-stuff-done/ 2>/dev/null || echo "Directories differ or get-shit-done/ does not exist"
   ```

b. If `get-shit-done/` exists:
   - For each file in `get-shit-done/` that does NOT exist in `get-stuff-done/`: copy it to `get-stuff-done/`
   - For each file in `get-shit-done/` that ALSO exists in `get-stuff-done/`: the `get-shit-done/` version is upstream's latest (from cherry-picks). Replace the `get-stuff-done/` version with it.
   - New directories in `get-shit-done/` (like `workflows/` files that are new): create them under `get-stuff-done/`
   - After merging: `rm -rf get-shit-done/`

c. If `get-shit-done/` does NOT exist (cherry-picks may have modified `get-stuff-done/` directly if git detected the rename): skip this step.
