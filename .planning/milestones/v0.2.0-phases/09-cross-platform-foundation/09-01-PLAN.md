  <action>
1. Add pathe as a production dependency: `bun add pathe`

2. Create `src/platform/detect.js` - Platform detection singleton:
   - Detect OS via `os.platform()`: win32, darwin, linux
   - Detect shell: check `process.env.SHELL` (Unix) and `process.env.ComSpec` (Windows)
   - Classify shell: isBash, isZsh, isPowerShell, isCmd
   - Detect environment: isMingw (check MSYSTEM, MINGW_PREFIX env vars), isWSL (check /proc/version for Microsoft or WSL_DISTRO_NAME), isGitBash (MSYSTEM + EXEPATH containing Git)
   - Detect Node.js version and validate >= 20 LTS (warn but don't block on older)
   - Detect git availability via `which git` or `where git` check
   - Cache results as singleton (detect once, reuse)
   - Export `detectPlatform()` which returns the cached result object

3. Create `src/platform/paths.js` - Path utility wrapping pathe:
   - Re-export join, resolve, normalize, dirname, basename, extname from pathe
   - Add `gsdHome()`: returns GSD_HOME env var or `path.join(os.homedir(), '.gsd')`
   - Add `claudeHome()`: returns CLAUDE_CONFIG_DIR env var or `path.join(os.homedir(), '.claude')`
   - Add `toForwardSlash(p)`: ensures forward slashes (for display/logging)
   - Add `expandTilde(p)`: expand ~ to os.homedir()
   - All functions produce forward-slash paths via pathe

4. Create `src/platform/terminal.js` - Terminal capability detection:
   - Detect color support levels: 0=none, 1=basic(16), 2=256color, 3=truecolor
   - Use COLORTERM, TERM, FORCE_COLOR env vars
   - Detect Windows Terminal (WT_SESSION), ConEmu (ConEmuTask), VS Code (TERM_PROGRAM=vscode), iTerm2 (TERM_PROGRAM=iTerm.app), Alacritty, Kitty, Hyper
   - Detect Unicode support: Windows Terminal and modern terminals = true, legacy Windows Console Host = false, Linux framebuffer (TERM=linux) = false
   - Report terminal dimensions: process.stdout.columns/rows with defaults 80x24
   - Export `detectTerminal()` returning cached capability object

5. Create `src/platform/index.js` - Unified re-export:
   - Export `platform` (result of detectPlatform())
   - Export `gsdPaths` (all path functions)
   - Export `terminal` (result of detectTerminal())
   - Export individual detect functions for direct use

Important: Follow existing codebase conventions (CommonJS require/module.exports, no TypeScript, same code style as src/config/ and src/theme/).
