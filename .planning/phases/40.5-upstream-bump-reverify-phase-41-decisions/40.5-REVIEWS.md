Wave 4 is verifying decisions in a CONTEXT.md whose count-of-decisions claim is itself drift between text and reality. The plans don't instruct the executor to either (a) silently accept the count or (b) propose an A-07 amendment fixing the stale wording. This is exactly the kind of drift D-05 contemplates ("cited symbol/API/line behavior changed") at the meta level. **Plan 04 Task 04-01** should add an instruction to flag this for amendment review.

- **MEDIUM — Plan 03 Task 03-02 duplicates RESEARCH.md §1 work without referencing it.** RESEARCH.md §1 already executed the static path cross-check across 21 overlay paths against the v1.38.2 npm tarball (zero collisions). Task 03-02 instructs the executor to "enumerate fork overlay files" and "derive the corresponding upstream path the compose pipeline targets" without telling them to start from RESEARCH.md's existing table. The right shape: "verify RESEARCH.md §1's table still holds at v1.38.5 (changes from 1.38.2..1.38.5 are 5 commits / 2 files per RESEARCH.md §1 reconciliation), updating only the rows where the v1.38.5 tree differs." As written, the executor redoes the analysis from scratch, increasing both time and the chance of inconsistency with the prior research output.

- **MEDIUM — Plan 05's 999.3 Path B updates PROJECT.md/ROADMAP.md outside any task's `files_modified` boundary.** Plan 05 Task 05-03 step 4 (Path B / NOT-A-GAP close) says "Update PROJECT.md / ROADMAP.md backlog 999.3 entry to status `closed-not-a-gap`". Task 05-03 has empty `<files>` (it's a checkpoint task). Task 05-05's atomic commit lists only `STATE.md`, `40.5-VERIFICATION.md`, `40.5-README.md`. Either Task 05-03 needs explicit `<files>` declaring PROJECT.md/ROADMAP.md or Task 05-05's `files_modified` needs to expand. Without this, the close-NOT-A-GAP update either gets orphaned (uncommitted) or smuggled into Wave 5's commit unannounced.

- **MEDIUM — External-action version refresh tooling is unspecified.** Plan 04 Tasks 04-02..04-05 instruct the executor to "consult the action's GitHub releases" for `gitleaks-action@v2`, `osv-scanner-action@v2`, `step-security/harden-runner@v2`, `markdownlint-cli2@0.22.0`, hyperfine. The verification command isn't specified — `gh release list -R <owner>/<repo>` is the obvious tool but the plan doesn't say so. If the executor lacks `gh` CLI auth or WebFetch, this step degrades to "presumed unchanged" silently. Should specify a fallback (e.g., "if no release-listing tool available, mark `unchanged-presumed` and surface the limitation in VERIFICATION.md").

- **LOW — Plan 04 Task 04-06 Target 3 grep regex is awkwardly escape-quoted.** The command `grep -n -E "UI\|interface\|frontend\|..."` uses ERE-escaped pipes (`\|` = literal `|` in ERE) to find the upstream source line containing the bug pattern. This works for detecting the broken regex but won't match a fixed `\b(UI|interface|...)\b` form. Path B detection (already-fixed-by-upstream) relies on "pattern not found at all → inspect surrounding context" which is fine but could be made cleaner with two grep commands (one for broken, one for boundary-anchored fixed). Minor; the workflow degrades gracefully.

- **LOW — Several acceptance-criterion grep patterns are whitespace-brittle.** E.g., Plan 04 Task 04-07 uses `grep -cE '\| (unchanged|drifted|amended) \|' returns 24` — depends on exactly one space between pipe and word. If a row has `|  unchanged  |` (two spaces) or `| unchanged|` (no trailing space), the count fails. Same for the `<pending>` check. Markdown editors / formatters can rewrite tables. Lower risk because the executor is writing the file in the same task chain that checks it, but it's a brittle invariant.

- **LOW — Plan 02 Task 02-02 procedure assumes a bare clone exists at a known path.** The diff method "preferred" path is `git -C <bare-clone-dir> diff v1.34.2..v1.38.5 -- hooks/gsd-check-update.js` with no instruction on how to obtain or locate the bare clone. The npm-pack fallback is acknowledged but the bare clone setup is left to the executor. A one-line `git clone --bare https://github.com/gsd-build/get-shit-done.git /tmp/gsd-bare && cd /tmp/gsd-bare && git fetch --tags` (or equivalent) command would unblock this without forcing fallback.
