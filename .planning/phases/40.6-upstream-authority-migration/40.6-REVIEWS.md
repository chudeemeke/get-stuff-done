## Blocking Findings To Address Before Execution

### HIGH: Manifest Boundary Is Underspecified

`.planning/upstream-authority.json` may be unsafe if migrated scripts run from a
published package where `.planning` is excluded. The plans must specify whether the
manifest is maintainer/build-time only or package-included runtime data.

Required amendment:

- Treat `.planning/upstream-authority.json` as the planning/build-time source of truth.
- For any runtime or package-shipped code that needs authority metadata, generate or
  include a package artifact explicitly and verify it in pack/install testing.

### HIGH: Package Integrity And Provenance Evidence Is Missing

Plan 02 must not inspect the package layout without also recording the exact package
artifact being trusted.

Required amendment:

- Capture `dist.integrity`, `dist.shasum`, `dist.tarball`, `license`, `scripts`, and
  release/tag correlation for `@opengsd/gsd-core@1.5.0`.
- Compare against the current installed legacy package layout, not only remembered
  assumptions.

### HIGH: Plan 03 Does Not Define Packaged Output Behavior

The migration may pass source tests while failing in `dist/` or the packed package.

Required amendment:

- Decide whether `dist/**` is regenerated and committed during Plan 03.
- If not regenerated in Plan 03, Plan 04 must regenerate and verify it before marking
  Phase 40.6 complete.
