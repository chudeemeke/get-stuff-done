## Hard Fails

Any of these fail Phase 40.6:

- Active dependency remains `get-shit-done-cc`.
- Active tooling queries `get-shit-done-cc` as the upgrade authority.
- Implementation tracks `@opengsd/gsd-core@latest`, `next`, canary, or dev tags.
- Package layout is assumed without temp install/pack evidence.
- Package integrity/provenance metadata is not recorded before layout trust.
- Package-shipped code depends on `.planning/` existing.
- Packed artifact / scratch consumer install is not verified.
- Global installed GSD is patched as part of migration debugging.
- Phase 41 is resumed before Open GSD compose/override/update surfaces are verified.

## Allowed Legacy References

Legacy strings may remain only when clearly historical or evidentiary:

- Archived milestones.
- Old research files.
- This Phase 40.6 research and ADR.
- Comments explaining migration history.
- Explicit deprecation/legacy fields in `.planning/upstream-authority.json`.

Any active script or active config retaining a legacy reference must have a reason in
`40.6-VERIFICATION.md`.
