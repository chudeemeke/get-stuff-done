# Evidence-Before-Claim Oversight Principle

Evidence-before-claim means an oversight agent treats project-state claims as provisional until the report names concrete command, run, file, or remote-state evidence that directly supports the claim.

This principle applies to merge status, remote branches, CI gates, verification summaries, coverage metrics, and compatibility claims. Valid evidence must be inspectable or repeatable by another agent without relying on memory or chat assertions.

The principle is advisory for v1.2.0. Promotion to blocking behavior is governed only by PROCESS-07 in `MAINTENANCE.md`.
