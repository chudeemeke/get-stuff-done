**classifyCommit function (RED phase):**

Add `classifyCommit(subject, files)` to `get-stuff-done/bin/lib/sync.cjs`. The function is pure (no git calls, no cwd). Classification precedence (from CONTEXT.md):

1. Security keywords anywhere in subject: `/\b(security|cve|vuln|exploit|xss|sqli|rce|injection)\b/i` => `{ type: 'security', confidence: 'high' }`
2. Breaking change marker: `/^[a-z]+(\(.+\))?!:/` or `BREAKING CHANGE` in subject => `{ type: 'breaking', confidence: 'high' }`
3. Conventional commit prefix: `/^(feat|fix|refactor|docs|chore|test|perf|style)(\(.+\))?:/i` => high confidence. Normalize: test -> chore, perf -> refactor, style -> chore
4. File-path heuristics (fallback when no prefix):
   - docs/ or .md files (without bin/hooks) => `{ type: 'docs', confidence: 'medium' }`
   - tests/ or .test. or .spec. files (without bin/hooks) => `{ type: 'chore', confidence: 'medium' }`
   - bin/ or hooks/ files => `{ type: 'feat', confidence: 'low' }`
5. Nothing matched => `{ type: 'other', confidence: 'low' }`

Files parameter: array of objects with `path` property (matches existing getFilesForCommit output shape).

Export classifyCommit in module.exports.
