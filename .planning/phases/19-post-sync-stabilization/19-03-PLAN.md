  <action>
Migrate all 17 instances of `delete require.cache[require.resolve('../src/platform/detect')]` in platform.test.js to use the exported internal functions instead.

**Pattern to follow (from platform-internal.test.js, already proven to work):**

BEFORE (re-require pattern, breaks coverage):
```javascript
delete require.cache[require.resolve('../src/platform/detect')];
const { detectPlatform: dp, clearCache: cc } = require('../src/platform/detect');
cc();
const platform = dp();
expect(platform.shell).toBe('pwsh');
```

AFTER (direct internal call, coverage correct):
```javascript
const result = _detectShell();
expect(result.shell).toBe('pwsh');
```

**Step-by-step approach:**

1. At the top of platform.test.js where detect.js is required, add the _detect* exports to the destructuring:
   ```javascript
   const { detectPlatform, clearCache, _detectShell, _detectEnvironment, _detectNodeVersion, _detectGit } = require('../src/platform/detect');
   ```

2. For each of the 17 `delete require.cache` blocks, determine which _detect* function the test is exercising based on what it asserts:
   - Tests asserting `platform.shell` or `platform.shellVersion` -> use `_detectShell()`
   - Tests asserting `platform.environment` or `platform.isWSL` etc -> use `_detectEnvironment()`
   - Tests asserting `platform.nodeVersion` -> use `_detectNodeVersion()`
   - Tests asserting `platform.gitVersion` or `platform.gitAvailable` -> use `_detectGit()`

3. For each test:
   a. Remove the `delete require.cache` line
   b. Remove the re-require line (`const { detectPlatform: dp ... } = require(...)`)
   c. Remove the `cc()` / `clearCache()` call if present
   d. Replace `dp()` or `detectPlatform()` call with the appropriate `_detect*()` call
   e. Update assertions to match the return shape of the internal function:
      - `_detectShell()` returns `{ shell, shellVersion }` (subset of full platform object)
      - `_detectEnvironment()` returns `{ environment, isWSL, isDocker, isCI, isSSH, ciName }` (subset)
      - `_detectNodeVersion()` returns `{ nodeVersion }` (subset)
      - `_detectGit()` returns `{ gitAvailable, gitVersion }` (subset)

4. Keep all mockPlatform()/restorePlatform patterns and env variable mocking -- those still work. Only the module re-loading is being replaced.

5. **Exception:** If any test specifically tests the `detectGit()` error catch path (lines 183-187 of detect.js) by mocking execSync, that test may still need re-require because execSync is destructured at module load time. Leave those tests as-is (maximum 1-2 tests). The goal is to migrate the 17 process.platform/env-based tests, not the execSync mocking tests.

**Important:** Read platform-internal.test.js first to see the proven pattern for each _detect* function. Use it as a reference for assertion shapes and mock patterns.

**Important:** After migration, verify that each test still tests the same condition (same platform mock, same env vars, same expected output). The test LOGIC must not change -- only the mechanism for invoking detect.js changes.
