**Step 1: Create src/config/schema.js**

Define an AJV schema for `.planning/config.json` based on the actual file content AND gsd-tools.js loadConfig() structure. The real `.planning/config.json` has these keys (read it to confirm):

Top-level flat keys:
- `model_profile` -- string, one of ['quality', 'balanced', 'budget']
- `commit_docs` -- boolean
- `search_gitignored` -- boolean
- `branching_strategy` -- string, one of ['none', 'phase', 'milestone']
- `phase_branch_template` -- string
- `milestone_branch_template` -- string
- `research` -- boolean
- `plan_checker` -- boolean
- `verifier` -- boolean
- `parallelization` -- boolean or object with `enabled` field

Nested sections from gsd-tools.js loadConfig():
- `planning` -- object with `commit_docs` (boolean), `search_gitignored` (boolean)
- `git` -- object with `branching_strategy`, `phase_branch_template`, `milestone_branch_template`
- `workflow` -- object with `research` (boolean), `plan_check` (boolean), `verifier` (boolean)

Phase 10 sections (CRITICAL -- these exist in the real config file and MUST be in the schema):
- `memory` -- object with:
  - `enabled` (boolean)
  - `location` (string)
  - `curation` (string, enum: ['auto', 'manual'])
  - `staleness_check` (boolean)
- `effort` -- object with:
  - `default` (string)
  - `agents` (object -- keys are agent names, values are strings like 'medium', 'high')
- `teams` -- object with:
  - `enabled` (boolean)
  - `experimental_flag` (string)
  - `oversight` (object with `default` boolean and `per_workflow` object)
  - `soft_cap` (number)

The schema MUST have `additionalProperties: false` at the top level to catch typos, but the memory/effort/teams sections MUST be fully specified with their actual sub-properties. Using bare `{ type: 'object' }` without properties would pass validation but miss structural errors within those sections.

Use AJV (already in deps). Export the schema object and a validate function:

```javascript
const Ajv = require('ajv');

const planningConfigSchema = {
  $schema: 'http://json-schema.org/draft-07/schema#',
  type: 'object',
  properties: {
    // Flat keys (gsd-tools.js loadConfig)
    model_profile: { type: 'string', enum: ['quality', 'balanced', 'budget'] },
    commit_docs: { type: 'boolean' },
    search_gitignored: { type: 'boolean' },
    branching_strategy: { type: 'string', enum: ['none', 'phase', 'milestone'] },
    phase_branch_template: { type: 'string' },
    milestone_branch_template: { type: 'string' },
    research: { type: 'boolean' },
    plan_checker: { type: 'boolean' },
    verifier: { type: 'boolean' },
    parallelization: {
      oneOf: [
        { type: 'boolean' },
        { type: 'object', properties: { enabled: { type: 'boolean' } }, required: ['enabled'], additionalProperties: false }
      ]
    },
    // Nested sections (gsd-tools.js loadConfig alternate format)
    planning: {
      type: 'object',
      properties: {
        commit_docs: { type: 'boolean' },
        search_gitignored: { type: 'boolean' }
      },
      additionalProperties: false
    },
    git: {
      type: 'object',
      properties: {
        branching_strategy: { type: 'string', enum: ['none', 'phase', 'milestone'] },
        phase_branch_template: { type: 'string' },
        milestone_branch_template: { type: 'string' }
      },
      additionalProperties: false
    },
    workflow: {
      type: 'object',
      properties: {
        research: { type: 'boolean' },
        plan_check: { type: 'boolean' },
        verifier: { type: 'boolean' }
      },
      additionalProperties: false
    },
    // Phase 10 sections -- fully specified with real sub-properties
    memory: {
      type: 'object',
      properties: {
        enabled: { type: 'boolean' },
        location: { type: 'string' },
        curation: { type: 'string', enum: ['auto', 'manual'] },
        staleness_check: { type: 'boolean' }
      },
      additionalProperties: false
    },
    effort: {
      type: 'object',
      properties: {
        default: { type: 'string' },
        agents: { type: 'object' }  // keys are dynamic agent names, values are effort levels
      },
      additionalProperties: false
    },
    teams: {
      type: 'object',
      properties: {
        enabled: { type: 'boolean' },
        experimental_flag: { type: 'string' },
        oversight: {
          type: 'object',
          properties: {
            default: { type: 'boolean' },
            per_workflow: { type: 'object' }  // keys are workflow names, values are booleans
          },
          additionalProperties: false
        },
        soft_cap: { type: 'number', minimum: 1 }
      },
      additionalProperties: false
    }
  },
  additionalProperties: false
};

const ajv = new Ajv({ allErrors: true });
const validate = ajv.compile(planningConfigSchema);

function validatePlanningConfig(config) {
  const valid = validate(config);
  if (!valid) {
    return { ok: false, errors: validate.errors.map(e => {
      const path = e.instancePath || '';
      return `${path}: ${e.message || e.keyword}`;
    })};
  }
  return { ok: true };
}

module.exports = { planningConfigSchema, validatePlanningConfig };
```

**Step 2: Create bin/validate-configs.js**

Standalone script (NOT a bin entry, just in the files array). Structure:

```javascript
#!/usr/bin/env node
'use strict';

// Escape hatch (CONTEXT locked decision)
if (process.env.SKIP_CONFIG_VALIDATION === '1') {
  process.stderr.write('Warning: Config validation skipped (SKIP_CONFIG_VALIDATION=1)\n');
  process.exit(0);
}

const fs = require('fs');
const path = require('path');

const PROJECT_ROOT = path.resolve(__dirname, '..');

// Validator registry
const validators = {
  '.planning/config.json': validatePlanningConfigFile,
  '.planning/ROADMAP.md': validateMarkdownSections('ROADMAP', ['## Milestones', '## Phases', '## Progress']),
  '.planning/STATE.md': validateMarkdownSections('STATE', ['## Current Position', '## Session Continuity']),
  '.planning/PROJECT.md': validateMarkdownSections('PROJECT', ['## What This Is', '## Requirements', '## Context']),
  'package.json': validatePackageJson,
};
```

Implement each validator:
1. **validatePlanningConfigFile:** Read file, JSON.parse, call validatePlanningConfig from schema.js. Check for conflict markers.
2. **validateMarkdownSections(name, requiredSections):** Returns a function that reads the markdown file and checks each required section exists via regex. Also check for conflict markers (`<<<<<<<`, `=======`, `>>>>>>>`).
3. **validatePackageJson:** Read and parse package.json. Verify `files` array entries exist on disk (reuse logic similar to scripts/check-parity.js). Check for conflict markers.

Each validator returns `{ ok: true }` or `{ ok: false, errors: ['message1', ...'] }`.

Output format:
- Per-file: `PASS file` to stdout, or `FAIL file: error1, error2` to stderr
- Summary line: `N passed, M failed` to stdout
- Exit 0 if all pass, exit 1 if any fail
- Support `--quiet` flag: only print failures and summary

The script should gracefully handle missing files (e.g., if .planning/ does not exist, that is a FAIL for those validators, not a crash).
