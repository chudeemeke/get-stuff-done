1. Get the list of commits:
   ```bash

   ```

2. For each commit hash (in chronological order):
   ```bash
   git cherry-pick --no-commit <hash>
   ```

3. If cherry-pick conflicts:
   - Classify conflict type per the 4-step triage model:
     - **Mechanical** (branding/URLs/package names): Auto-resolve using branding-map.json patterns. Do not count toward threshold.
     - **Cosmetic** (whitespace/import order): Resolve, prefer upstream. Do not count.
     - **Semantic** (different logic): Requires judgment. Counts toward threshold.
     - **Structural** (file moved/reorganized/deleted): Highest risk. Counts toward threshold.
   - If 3+ semantic/structural conflicts in this batch: pause, commit clean work, document unresolved in approach-comparison.md, continue to next commit.
   - For protected files staged for deletion: `git restore --staged <file>`
   - For protected files with content conflicts: resolve per priority rule (fork wins on UX, upstream wins on architecture)
