<task type="auto">
  <name>Task 1: Create input validation module</name>
  <files>src/validation/index.js</files>
  <action>
Create src/validation/index.js with three exported validation functions following allowlist-based patterns from the research document:

1. `validateGitSHA(sha)` - Validates git commit SHA format:
   - Must be a string
   - Must match `/^[0-9a-f]{40}$/i` (exactly 40 hex chars, case-insensitive)
   - Also accept short SHAs: `/^[0-9a-f]{7,40}$/i` (7-40 hex chars)
   - Throw Error with message: `Invalid git SHA format: ${sha}` on failure
   - Return the validated sha on success

2. `validateBranchName(branch)` - Validates git branch name:
   - Must be a non-empty string
   - Must match `/^[a-zA-Z0-9][a-zA-Z0-9/_.-]*$/` (allowlist: alphanumeric start, then alphanumeric + hyphens, underscores, forward slashes, dots)
   - Must not contain `..` (git ref traversal)
   - Must not end with `.lock` (git restriction)
   - Must not contain `@{` (git reflog syntax)
   - Maximum length 255 characters
   - Throw Error: `Branch name contains invalid characters: ${branch}` on pattern failure
   - Throw Error: specific messages for .., .lock, @{, and length violations
   - Return the validated branch on success

3. `validateConfigPath(userPath, allowedBaseDir)` - Validates config file paths against traversal:
   - Must be a string
   - Apply decodeURIComponent() before validation (catch URL-encoded traversal)
   - Resolve to absolute path with path.resolve()
   - Verify resolved path starts with the resolved allowedBaseDir using startsWith()
   - On Windows, normalize both paths to forward slashes before comparison
   - Throw Error: `Path traversal detected: ${userPath}` on failure
   - Return the resolved, validated path on success

Use `const path = require('path');` for path operations. Export all three functions via module.exports.

Do NOT use denylists. All validation uses allowlists (explicit safe patterns).
  </action>
