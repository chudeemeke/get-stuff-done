<task type="auto">
  <name>Task 2: Create config loader module</name>
  <files>src/config/ConfigLoader.js</files>
  <action>
Create ConfigLoader.js that loads, parses, and validates config from ~/.gsd/config.json.

Functions to export:
1. getConfigPath(): Returns config path, respects GSD_CONFIG_PATH env var
   - Default: path.join(os.homedir(), '.gsd', 'config.json')
   - Override: process.env.GSD_CONFIG_PATH if set

2. loadConfig(): Loads and returns validated config
   - If file missing: create default config with JSON5 comments, return defaults
   - If file exists: read with fs.readFileSync, parse with JSON5.parse
   - Validate with validateConfig from ConfigSchema
   - On parse error: throw with filename and position
   - On validation error: throw with field and reason

3. getDefaults(): Returns default config object
   - version: 1
   - working_context: 100000
   - chrome: false
   - dangerous_skip_permissions: false

Default config file content (written as JSON5 with comments):
```json5
{
  // Config version (for future migrations)
  version: 1,

  // Your usable context in tokens before auto-compaction triggers
  // GSD adds a buffer internally for the compaction process
  working_context: 100000,

  // Enable Chrome browser integration (--chrome flag)
  chrome: false,

  // Skip all permission prompts (dangerous - use with caution)
  dangerous_skip_permissions: false,
}
```

Use os.homedir() for tilde expansion (Node.js does not expand ~ natively).
  </action>
  <verify>
```bash
# Test config loading
node -e "const { loadConfig, getConfigPath } = require('./src/config/ConfigLoader'); console.log('Path:', getConfigPath()); console.log('Config:', loadConfig());"
```
  </verify>
  <done>ConfigLoader.js exports loadConfig and getConfigPath. Missing config auto-creates with comments. Invalid config shows clear error.</done>
</task>

<task type="auto">
  <name>Task 4: Update statusline to use config for thresholds</name>
  <files>hooks/gsd-statusline.js</files>
  <action>
Modify gsd-statusline.js to read thresholds from config instead of hardcoded values.

1. Import ConfigLoader at top:
   ```javascript
   const { loadConfig } = require('../src/config/ConfigLoader');
   ```

2. Calculate dynamic thresholds from working_context:
   - The thresholds are fractions of the autocompact trigger point
   - autocompact_pct = ((working_context + 20000) / 200000) * 100
   - Green: 0 to 0.5 * autocompact_pct (50% of threshold)
   - Yellow: 0.5 to 0.75 * autocompact_pct (75% of threshold)
   - Orange: 0.75 to 0.875 * autocompact_pct (87.5% of threshold)
   - Red: above 0.875 * autocompact_pct

   Example with working_context=100000 (autocompact at 60%):
   - Green: 0-30% used
   - Yellow: 30-45% used
   - Orange: 45-52.5% used
   - Red: 52.5%+ used

3. Replace hardcoded thresholds (currently 50, 65, 80) with calculated values

4. Handle config load errors gracefully:
   - Wrap loadConfig() in try/catch
   - On error: use default thresholds (50, 65, 80) as fallback
   - Do NOT crash the statusline on config error

5. Keep existing statusline output format (model, task, dir, context bar)
