<task type="auto">
  <name>Task 1: Add installation detection and metadata functions</name>
  <files>bin/install.js</files>
  <action>
Add four helper functions to `bin/install.js` after the existing `createSymlink` function (around line 252):

1. `detectInstallationType(targetDir)` - async function that:
   - Checks key directories: `commands/gsd`, `get-stuff-done`, `agents`, `hooks`
   - Uses `fs.promises.lstat()` (NOT `fs.stat()`) to check `isSymbolicLink()`
   - Returns `'link'` if ANY checked path is a symlink
   - Returns `'copy'` if paths exist but none are symlinks
   - Returns `null` if no existing installation found

2. `readInstallMetadata(targetDir)` - sync function that:
   - Reads `{targetDir}/get-stuff-done/.install-meta.json`
   - Returns parsed JSON object or `null` on error/missing

3. `writeInstallMetadata(targetDir, installType, version)` - sync function that:
   - Creates metadata object: `{ version, installType, installedAt: ISO string, platform: process.platform }`
   - Writes to `{targetDir}/get-stuff-done/.install-meta.json`
   - Returns the metadata path

4. `determineInstallMode(targetDir, hasLinkFlag, useLinks)` - async function that:
   - If `hasLinkFlag` is true, return `useLinks` (user explicitly chose)
   - Try `readInstallMetadata()` first (fast, reliable)
   - Fallback to `detectInstallationType()` if no metadata
   - Log what was detected: `"  Detected {type} installation, matching mode"`
   - Return boolean (true = use links, false = use copies)
   - Default to `false` (copies) if no existing installation

Use patterns from 03-RESEARCH.md. Key insight: `fs.lstat()` is required because `fs.stat()` follows symlinks and won't detect them.
  </action>

  </verify>

Four functions exist in bin/install.js: detectInstallationType, readInstallMetadata, writeInstallMetadata, determineInstallMode. Syntax check passes.
  </done>
