    **GREEN PHASE -- Implement pipeline stages in scripts/compose.js:**

    Add the 5 pipeline stage functions to scripts/compose.js (which already has branding functions from Plan 01). Each function takes a pipeline state object and returns a modified state object.

    **Pipeline state shape:**
    ```javascript
    {
      manifest: ManifestEntry[],   // file entries flowing through stages
      branding: Object,            // parsed branding.json config
      features: Object,            // parsed features.json config
      warnings: string[],          // accumulated warnings
      meta: {                      // .install-meta.json data
        upstream_version, overlay_version, composed_at,
        features_disabled, overrides_applied, branding_rules_applied
      },
      overlayFiles: ManifestEntry[], // overlay files (additive, merged in MERGE stage)
      upstreamDir: string,          // resolved upstream path
      overlayDir: string,           // resolved overlay path
      overridesDir: string,         // resolved overrides path
    }
    ```

    **ManifestEntry shape:**
    ```javascript
    { sourcePath, destPath, content: null|string, action: 'copy'|'brand'|'override'|'overlay'|'skip', stage: string }
    ```

    1. `resolve(upstreamDir, overlayDir, overridesDir)`:
       - Validate upstream structure against EXPECTED_UPSTREAM_PATHS (from research: bin/install.js, commands/gsd, get-shit-done, get-shit-done/bin/gsd-tools.cjs, agents, hooks/dist, hooks/dist/gsd-check-update.js, package.json). Fail fast on first missing path with descriptive error + preview-update hint.
       - Walk upstream directory recursively, create ManifestEntry for each file (action: 'copy', content: null).
       - Walk overlay directory (if exists, may be empty -- handle gracefully).
       - Walk overrides directory (if exists, may be empty).
       - Detect collisions: overlay file matching upstream relative path (not in overrides) is an error. Error message names both sources and shows: `mv overlay/X overrides/X` + `echo '# Override: X' > overrides/X.REASON.md`.
       - Load and validate branding.json (use validateBrandingConfig from Plan 01).
       - Load features.json.
       - Read upstream_version from node_modules/get-shit-done-cc/package.json.
       - Read overlay_version from project root package.json.
       - Return pipeline state.
       - Normalize all relative paths with forward slashes: `.replace(/\\/g, '/')`.

    2. `filter(state)`:
       - Phase 30 stub: return state unchanged. Comment that Phase 31 implements full feature flag filtering.
       - Still mark features_disabled as empty array in state.meta.

    3. `override(state)`:
       - Phase 30 stub: return state unchanged. Comment that Phase 31 implements full override replacement.
       - Still mark overrides_applied as empty array in state.meta.

    4. `brand(state)`:
       - Sort substitutions by length descending (sortSubstitutions from Plan 01).
       - For each manifest entry: if shouldBrandFile(destPath) returns true, read file content, apply applyBrandingToContent, check if content actually changed. If changed: set entry.content = modified, entry.action = 'brand'. If not changed: leave as 'copy'.
       - Track branding_rules_applied count (number of files where substitutions produced changes).
       - For any substitution rule that matched zero files across the entire manifest: add warning "Branding rule '{from}' matched zero files (rule may be stale)".
       - Set state.meta.branding_rules_applied.

    5. `merge(state, distDir, opts)`:
       - If `opts.dryRun`: skip all file writes, return state with summary data.
       - If `opts.diff`: compare what would be written against current dist/ contents. Report added/removed/changed filenames. No content-level diffs. Return state.
       - Normal mode: delete distDir entirely (fs.rmSync recursive), then recreate.
       - For each manifest entry (not 'skip'): if content is non-null, writeFileSync. If content is null, copyFileSync.
       - Copy overlay files to dist/ (additive merge).
       - Copy upstream LICENSE to dist/ unmodified.
       - If preserveUpstreamCredit: write CREDITS.md to dist/ root (use generateCredits from Plan 01).
       - Write .install-meta.json to dist/ root.

    6. `compose(opts)` -- Orchestrator function:
       - Resolve project root paths (upstream, overlay, overrides, dist).
       - Call resolve -> filter -> override -> brand -> merge in sequence.
       - Return final state (for summary printing).

    7. Add CLI entry point at the bottom of compose.js:
       - Parse process.argv for --dry-run, --diff, --verbose flags.
       - Only run CLI when file is executed directly (not require'd): `if (require.main === module)`.
       - Verbose mode: print `[RESOLVE]`, `[FILTER]`, `[OVERRIDE]`, `[BRAND]`, `[MERGE]` stage progress with counts to stdout.
       - Default mode: silent during work, print summary at end.
       - Summary format (per CONTEXT locked decision): indented key-value pairs showing upstream files, overlay files, branding rules applied, overrides (0), warnings, output size.
       - Dry-run banner: "DRY RUN -- no files written" before summary.
       - Errors go to stderr (process.stderr.write), then process.exit(1).

    Export all stage functions and compose() for testing: `module.exports = { resolve, filter, override, brand, merge, compose, ... }`.

    - .install-meta.json with correct data
    - CREDITS.md (if preserveUpstreamCredit: true)
    - LICENSE from upstream
    - No overlay code files (overlay/ is empty scaffold at this point)
