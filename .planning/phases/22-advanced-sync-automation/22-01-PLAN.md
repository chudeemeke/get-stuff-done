Add 4 new functions to sync.cjs between the supply chain scanner section and cmdSyncPreview. Add them to module.exports under a new comment section "// Selective sync filtering".

**1. filterCommitsByCategory(commits, filters)**

Pure function that filters enriched commits based on category inclusion/exclusion and individual SHA overrides.

Parameters:
- `commits` - Array of enriched commit objects (must have `hashShort`, `hash`, `classification.type`)
- `filters` - Object: `{ categories?: string[], excludeCategories?: string[], includeShas?: string[], excludeShas?: string[] }`

Precedence rules (per CONTEXT.md):
1. Individual SHA --exclude overrides everything (force exclude)
2. Individual SHA --include overrides category filters (force include)
3. If `categories` provided and non-empty, only those types included
4. If `excludeCategories` provided and non-empty, those types removed from the set
5. Both can combine: `--category feat,fix --exclude feat` gives only fix

Returns: `{ selected: commit[], excluded: Array<commit & { excludeReason: 'sha-excluded' | 'category-filtered' }> }`

**2. detectModules(files)**

Detects which bin/lib/*.cjs modules a commit's files touch.

Parameters:
- `files` - Array of `{ path }` objects (from getFilesForCommit or enriched files)

Pattern to match: `/(?:get-stuff-done\/)?bin\/lib\/([^/]+)\.cjs$/`

Returns: `string[]` - Module names (e.g., ['sync', 'state', 'core'])

**3. isCrossModule(modulesA, modulesB)**

Checks whether two sets of modules share any overlap or have unique entries.

Parameters: Two string arrays of module names.

Returns: `boolean` - true if the module sets are different (any module in A not in B or vice versa)

**4. detectFileOverlapDeps(commits)**

Detects file-overlap dependencies between chronologically ordered commits. If commit B modifies a file also modified by an earlier commit A, B may depend on A.

Parameters:
- `commits` - Array of objects with `{ hash, hashShort, files: Array<{path}> }` in chronological order

Returns: `Array<{ commit: string, dependsOn: string, reason: string, type: 'file-overlap', files: string[] }>`

Implementation: Build a Map of filePath -> earliest commit. For each subsequent commit touching the same file, record the dependency. Use the `hashShort` for commit identifiers in the output.

All 4 functions must use spawnGit() if they need any git operations (though these are pure data functions so they should not need git). Export all 4 in module.exports.

**Part B: Extend gsd-tools.cjs sync-preview case**

Update the `sync-preview` case (line 586-588) to parse filter flags from args:

```javascript
case 'sync-preview': {
  // Parse filter flags
  const categories = extractFlagValues(args, '--category');
  const excludeCategories = extractFlagValues(args, '--exclude');
  const includeShas = extractShaCandidates(args, '--include');
  const excludeShas = extractShaCandidates(args, '--exclude-sha');

  sync.cmdSyncPreview(cwd, args[1], {
    json: args.includes('--json'),
    categories: categories.length ? categories : undefined,
    excludeCategories: excludeCategories.length ? excludeCategories : undefined,
    includeShas: includeShas.length ? includeShas : undefined,
    excludeShas: excludeShas.length ? excludeShas : undefined,
  }, raw);
  break;
}
```

Add two small helper functions near the top of gsd-tools.cjs (after the existing helper functions, before the switch statement):

`extractFlagValues(args, flag)` - Finds `flag` in args, takes the next arg, splits by comma. Returns string[]. If flag not found, returns [].

`extractShaCandidates(args, flag)` - Same as extractFlagValues but validates each value matches `/^[0-9a-f]{7,40}$/i`. Ignores non-SHA values.

Note: The --exclude flag is used for BOTH category exclusion and SHA exclusion. Distinguish by format: values matching SHA pattern (7-40 hex chars) go to excludeShas; non-matching values go to excludeCategories. Handle the case where user passes `--exclude refactor` (category) vs `--exclude abc1234` (SHA). Better approach per RESEARCH.md: use `--exclude` for categories and `--exclude-sha` for individual SHA exclusion, with `--include` for individual SHA inclusion. This avoids ambiguity.

Update the help text in the `help` case and the `--help` case to include the new flags:
```

```
