<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repo suppression data to security gate | A PR can edit suppressions to hide vulnerabilities. |
| Security docs to maintainer action | Ambiguous policy can cause inconsistent handling of findings. |
| ESLint config to source review | Disabling security rules can hide unsafe code paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-02-01 | Tampering | `.planning/audits/suppressions.json` | mitigate | Strict AJV schema, no extra fields, required reviewer/date/reason, TTL failure. |
| T-41-02-02 | Repudiation | suppression review | mitigate | `reviewer`, `reviewedDate`, and `reReviewDate` required on every non-empty entry. |
| T-41-02-03 | Elevation of Privilege | eslint security config | mitigate | Keep production security rules enabled and document test-only exceptions. |
</threat_model>
