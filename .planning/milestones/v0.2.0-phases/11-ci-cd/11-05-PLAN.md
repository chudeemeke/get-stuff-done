  <action>
    Create scripts/check-parity.js -- a Node.js script that verifies all files declared in package.json "files" array exist in the project source.

    The script:
    1. Reads package.json and extracts the "files" array (currently: bin, commands, get-stuff-done, agents, hooks/dist, scripts, src)
    2. For each entry in "files":
       - If it is a directory: verify it exists and is non-empty
       - If it is a file: verify it exists
    3. Additionally checks that key distributable files exist:
       - bin/gsd.js (the launcher)
       - bin/install.js (the installer)
       - get-stuff-done/bin/gsd-tools.js (the CLI tools)
       - src/config/ConfigLoader.js
       - src/platform/detect.js
       - hooks/dist/ has at least the 3 hook files (after build:hooks)
         NOTE: hooks/dist/ is gitignored and built by prepublishOnly. The parity check should either:
         (a) Run build:hooks first then check, or
         (b) Check hooks/ source files instead of dist/, or
         (c) Skip hooks/dist/ with a note that it is generated
         Choose option (c): skip hooks/dist/ in CI parity check, note it is build-generated. Check hooks/ source files (gsd-check-update.js, gsd-statusline.js, pre-compact.js) instead.
    4. Reports results:
       - Lists each entry with PASS/FAIL status
       - Exits 0 if all pass, exits 1 if any fail
       - Outputs summary: "Parity check: X/Y passed"

    Script uses only Node.js built-ins (fs, path). No dependencies needed.

<task type="auto">
  <name>Task 2: Create GitHub Actions CI workflow and update ESLint config</name>
  <files>
    .github/workflows/ci.yml
    eslint.config.js
  </files>
  <action>
    **Create .github/workflows/ci.yml:**

    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      lint:
        name: Lint
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: oven-sh/setup-bun@v2
            with:
              bun-version: latest
          - run: bun install --frozen-lockfile
          - run: bun run lint

      test:
        name: Test (${{ matrix.os }})
        runs-on: ${{ matrix.os }}
        strategy:
          fail-fast: false
          matrix:
            os: [ubuntu-latest, macos-latest, windows-latest]
        steps:
          - uses: actions/checkout@v4
          - uses: oven-sh/setup-bun@v2
            with:
              bun-version: latest
          - name: Cache bun dependencies
            uses: actions/cache@v4
            with:
              path: ~/.bun/install/cache
              key: ${{ runner.os }}-bun-${{ hashFiles('bun.lock') }}
              restore-keys: |
                ${{ runner.os }}-bun-
          - run: bun install --frozen-lockfile
          - run: bun test --coverage

      parity:
        name: Source Parity Check
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - run: node scripts/check-parity.js
    ```

    Key design decisions:
    - fail-fast: false -- so all 3 OS results are visible even if one fails
    - Lint runs on ubuntu-only (ESLint is platform-independent, no need for 3x)
    - Tests run on all 3 OSes with matrix
    - Parity check runs on ubuntu-only (filesystem presence check, platform-independent)
    - Bun cache uses bun.lock hash for cache key
    - Uses --frozen-lockfile to ensure CI uses exact lockfile versions
    - No artifact uploads per CONTEXT.md decision
    - No branch protection per CONTEXT.md decision (signal only)
    - No secrets needed (all externals mocked in tests)

    **Update eslint.config.js:**

    Add a configuration block for test files that relaxes security rules likely to produce false positives on test mocks:

    ```javascript
    {
      files: ['tests/**/*.js'],
      rules: {
        'security/detect-non-literal-fs-filename': 'off',
        'security/detect-non-literal-require': 'off',
        'security/detect-child-process': 'off',
        'security/detect-object-injection': 'off',
      }
    }
    ```

    This is necessary because tests intentionally use dynamic paths (temp dirs), dynamic requires (mock modules), and child_process (running scripts under test). These are legitimate in test code, not security vulnerabilities.

    Keep the existing eslint config blocks unchanged. Append the test config block after the existing configuration.
  </action>
