<objective>
Close three gaps identified by Codex (GPT-5.4) cross-AI review of Phase 37 plan 01:

1. **Path traversal in manifest entries (HIGH, security):** removeGsdFiles() deletes path.join(targetDir, relPath) without verifying the resolved path stays within targetDir. A malicious/corrupt manifest entry like `../../.bashrc` escapes the target. Same vulnerability class as the original wipe incident.

2. **uninstall() public entrypoint untested (HIGH):** Plan 01 tests removeGsdFiles() (internal) but never the uninstall() function that calls it. The wiring between --uninstall and removeGsdFiles is unverified.

3. **Side-effect verification incomplete (MEDIUM):** The export test captures stdout via subprocess but does not assert stderr is also empty in the same check.

Purpose: Eliminate residual destructive-cleanup risk in the manifest-driven installer by hardening the path handling and testing the public uninstall entrypoint.

Output: Hardened removeGsdFiles with path containment + uninstall export and tests + strengthened side-effect verification.
</objective>
