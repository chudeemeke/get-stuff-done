    ```
        - returns null when preserveUpstreamCredit is false (BRAND-05)

      describe('integration: real upstream install.js branding')
        - branded install.js retains exactly 29 bare 'get-shit-done' path references (BRAND-02, BRAND-03)
        - branded install.js has zero 'get-shit-done-cc' occurrences (all replaced) (BRAND-01)

    ```

    Run tests -- they MUST fail (RED). Commit: `test(30-01): add failing branding engine tests`

    **GREEN PHASE -- Implement in scripts/compose.js:**

    Create scripts/compose.js with the following exported functions:

    1. `applyBrandingToContent(content, substitutions)` -- Takes file content string and sorted substitution array. Applies `replaceAll` for package-name and repo-URL substitutions. Uses word-boundary regex (`\b`) for short tokens like TACHES to prevent partial-word replacement. Returns modified content.

    2. `validateBrandingConfig(config)` -- Uses AJV to validate branding.json against the schema from the research (type: object, required: ['substitutions'], substitutions array with required from/to/scope, scope enum: ['text'], optional preserveUpstreamCredit boolean, additionalProperties: false). Throws descriptive error on invalid config.

    3. `sortSubstitutions(substitutions)` -- Sorts by `from` string length descending. This ensures most-specific patterns apply first (e.g., `glittercowboy/get-shit-done` before `get-shit-done-cc` before `TACHES`).

    4. `shouldBrandFile(relPath)` -- Returns false for binary extensions (.png, .svg, .gif, .ico, .woff, .woff2, .ttf, .eot), for paths starting with 'overlay/', for paths starting with 'overrides/', and for the file named LICENSE (case-insensitive). Returns true otherwise.

    5. `generateCredits(preserveUpstreamCredit)` -- When true, returns the CREDITS.md content string from the design spec: "This software is based on [GSD (Get Shit Done)](https://github.com/glittercowboy/get-shit-done) by TACHES." etc. When false, returns null.

    The file should use CJS (require/module.exports) matching all other scripts/ files in the project. Export all 5 functions plus any supporting schemas/constants needed by tests.

    Run tests -- they MUST pass (GREEN). Commit: `feat(30-01): implement branding engine`

    Key constraints:
    - Substitutions with `scope: "text"` only. Never implement `scope: "all"` (deprecated from original design).
    - The `from` patterns in branding.json are literal strings, not regex. The engine uses `replaceAll()` for most and word-boundary regex for short tokens (TACHES).
    - Binary file detection uses file extension only (not content sniffing).
    - AJV for schema validation -- do not hand-roll JSON validation.
