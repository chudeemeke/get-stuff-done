<objective>
Implement SSOT architecture for autocompact threshold and fix blink detection.

Philosophy (Apple/Anthropic approach):
- Built-in default lives in Claude Code (we don't control it)
- User override via ~/.gsd/THRESHOLD (optional)
- No fallback chains - either user set it or Claude Code's default is used

Output:
- ~/.gsd/THRESHOLD = optional user override
- If THRESHOLD exists → GSD sets env var
- If THRESHOLD missing → env var not set → Claude Code uses internal default
- Blink detection handles Windows Terminal correctly
</objective>

Key facts:
- User environment: Git Bash in Windows Terminal (WT_SESSION set)
- Claude Code has an internal default for autocompact threshold
- CLAUDE_AUTOCOMPACT_PCT_OVERRIDE env var overrides Claude's default
- If env var is NOT set, Claude Code uses its internal default
- THRESHOLD file is OPTIONAL - only for users who want to override

Architecture (simple):
```
~/.gsd/THRESHOLD exists?
  ├─ YES → Read value, export CLAUDE_AUTOCOMPACT_PCT_OVERRIDE
  └─ NO  → Don't set env var → Claude Code uses internal default
```

This is true SSOT: ONE source (THRESHOLD), no fallbacks, no hidden defaults.

    ```javascript
    // Terminal blink support detection
    function supportsBlinking() {
      const term = process.env.TERM || '';
      const termProgram = process.env.TERM_PROGRAM || '';

      // Windows Terminal: does NOT support ANSI blink despite xterm compatibility
      // Check FIRST because Git Bash sets TERM=xterm-256color in WT
      if (process.env.WT_SESSION) return false;

      // VS Code integrated terminal: no blink support
      if (termProgram === 'vscode') return false;

      // GNOME Terminal: no blink support
      if (termProgram === 'gnome-terminal') return false;

      // Known to support blink: native xterm, iTerm, Konsole
      if (term.includes('xterm') || termProgram === 'iTerm.app') return true;

      // Default: optimistically try blink
      return true;
    }
    ```

    Key: Check WT_SESSION BEFORE xterm check.
