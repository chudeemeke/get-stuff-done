  <action>
Create `get-stuff-done/bin/lib/sync.cjs` following the `commands.cjs` structural template (same require pattern, same output/error helpers from core.cjs).

The module exports these 4 commands plus internal helpers:

**cmdSyncPreview(cwd, range, options, raw):**
- Accepts range as "firstSHA..lastSHA" string. Parse into baseRef and targetRef.
- Validate that both SHAs exist using `execGit(cwd, ['cat-file', '-t', sha])`. If either missing, call `error('SHA not found -- run git fetch upstream main first')`.
- Check working tree is clean via `execGit(cwd, ['status', '--short'])`. If dirty, call `error('Working directory is dirty. Commit or stash changes before preview.')`.
- Call `getCommitsInRange(cwd, baseRef, targetRef)` to get commit list.
- For each commit, call `getFilesForCommit(cwd, sha)` to get file changes.
- Load protected paths from `.planning/sync/branding-map.json` via `loadProtectedPaths(cwd)`. Extract the `fork` field from each entry in `path_patterns`, `npm_patterns`, `github_patterns`, and `display_name_patterns`. Also include entries from `package_json_protected_fields` and `post_module_split_only` if present.
- For each file, call `isSensitivePath(filePath, protectedPaths)` to flag sensitive files.
- Call `assessConflictRiskByOverlap(cwd, commitFiles)` for basic file overlap heuristic.
- Call `computeEffortEstimate(cwd, commits.length)` for historical conflict rate from `.planning/sync/sync-manifest.json`.
- If `options.json` is true, output JSON matching the locked schema (see CONTEXT.md). Use `--no-color` for git commands in JSON path. The schema: `{ range, commits: [{hash, hashShort, subject, date, author, files: [{path, status, sensitive}], conflictRisk, securityFlags}], summary: {totalCommits, sensitivePathCount, highRiskCount, overlapRiskCount}, effortEstimate: {historicalConflictRate, estimatedConflicts, estimatedCleanCommits, dataPoints} }`.
- If `options.json` is false (default), output human-readable colorized terminal format: stat summary per commit, [!] markers on sensitive paths, conflict risk indicators, effort estimate section at bottom.
- For human-readable output, use `execGit(cwd, ['diff', '--color=always', '--stat', sha + '^..' + sha])` for each commit. Handle the edge case where sha^ fails on root commit by checking `execGit(cwd, ['rev-parse', sha + '^'])` first.

**cmdSyncCheckpointCreate(cwd, batchId, raw):**
- Validate batchId is provided, error if missing.
- Create annotated tag: `git tag -a sync-checkpoint-{batchId} -m "sync checkpoint before batch {batchId}" HEAD`. MUST include `-m` flag (Windows Git Bash hangs without it).
- Output `{ tag, sha: headSha, created: isoTimestamp }` via output().

**cmdSyncCheckpointList(cwd, raw):**
- List tags: `git tag -l 'sync-checkpoint-*'`.
- For each tag, get the SHA: `git rev-list -1 {tag}`.
- Output `{ checkpoints: [{tag, sha}], count }` via output().

**cmdSyncCheckpointCleanup(cwd, raw):**
- List all `sync-checkpoint-*` tags, delete each with `git tag -d`.
- Track deleted and failed arrays.
- Output `{ deleted, failed, count: deleted.length }` via output().

**Internal helpers (all exported for testability per bun coverage pitfall):**
- `getCommitsInRange(cwd, baseRef, targetRef)` - git log with format parsing
- `getFilesForCommit(cwd, sha)` - git diff-tree file list
- `loadProtectedPaths(cwd)` - reads branding-map.json, returns flat array of protected path strings
- `isSensitivePath(filePath, protectedPaths)` - string matching
- `assessConflictRiskByOverlap(cwd, commitFiles)` - git status dirty file overlap check
- `computeEffortEstimate(cwd, pendingCount)` - historical conflict rate from sync-manifest.json

Then register sync.cjs in `get-stuff-done/bin/gsd-tools.cjs`:
1. Add `const sync = require('./lib/sync.cjs');` with the other requires at the top.
2. Add switch cases:
   - `case 'sync-preview':` - parse range from `args[1]`, check for `--json` flag, call `sync.cmdSyncPreview(cwd, args[1], { json: args.includes('--json') }, raw)`.
   - `case 'sync-checkpoint':` - dispatch `args[1]` as subcommand to create/list/cleanup. For create, pass `args[2]` as batchId.
3. Update both help text blocks (the `--help` case and the `help` case) to include the new sync commands.
4. Update the error message in the default case if it lists available commands.

Do NOT strip ANSI codes in the human-readable path -- only use `--no-color` for the JSON output path.
