### Critical: gsd-tools.js validation require breaks post-install

`get-stuff-done/bin/gsd-tools.js` line 37:
```javascript
const { validateGitSHA, ... } = require('../../src/validation');
```

- **At source:** `../../src/validation` resolves to `<project>/src/validation/` -- works
- **Post-install:** `~/.claude/get-stuff-done/bin/gsd-tools.js` -> `~/.claude/src/validation/` -- DOES NOT EXIST
- **Impact:** MODULE_NOT_FOUND on every GSD command that invokes gsd-tools.js
- **Why tests pass:** Tests run from project root where relative path resolves correctly
- **Precedent:** Same class of bug Phase 13 fixed for hooks via esbuild bundling
- **Fix options:** (A) esbuild bundle gsd-tools.js like hooks, (B) installer copies src/ alongside get-stuff-done/, (C) inline validators into gsd-tools.js

## Gaps

### Critical (blocks ship)

**GAP-1: gsd-tools.js validation require path breaks post-install**

- `require('../../src/validation')` resolves at source but fails after installer copies to `~/.claude/`
- The installer copies `get-stuff-done/` but not `src/` to the install destination
- Every GSD command using gsd-tools.js would fail with MODULE_NOT_FOUND
- Same bug class as Phase 13 (hooks), same fix pattern available (esbuild bundling)
- **Affected files:** `get-stuff-done/bin/gsd-tools.js` (line 37), `bin/install.js` (copy logic)
- **Fix pattern:** esbuild bundle gsd-tools.js to inline src/validation (Phase 13 precedent), or have installer copy src/ to ~/.claude/src/, or inline validators into gsd-tools.js

**GAP-2: macOS/Linux manual testing not performed**

- Phase 9 cross-platform code is complete but never tested on physical macOS or Linux hardware
- CI matrix (Phase 11) runs on all 3 platforms and passes, covering functional correctness
- Manual testing needed for: installation (symlinks created correctly), launcher execution, hook execution, statusline rendering with Unicode/ANSI, permission failure fallback chain
- **Fix pattern:** Create automated integration tests or document manual test results from CI artifacts

**GAP-3: Platform module coverage below 95% WoW threshold**

- `src/platform/detect.js`: 100% functions, 66.67% lines
- `src/platform/terminal.js`: 100% functions, 93.50% lines
- **Fix pattern:** Add platform-specific test cases with mocked process.platform/env

**GAP-4: src/platform/index.js is dead code**

- Phase 9 created unified platform API re-export, zero consumers
- **Fix pattern:** Delete the file

**GAP-5: Agent teams not wired into workflows**

- Phase 10 created 4 team templates but no workflow reads teams.enabled
- **Fix pattern:** Add conditional logic to 4 workflows
